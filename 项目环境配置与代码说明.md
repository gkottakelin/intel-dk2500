# JetArm 项目环境配置、目录结构与代码说明

本文面向第一次接触本项目的开发者，目标是在一台新电脑上完成环境配置、运行无硬件测试，并理解 `project/` 中各目录和代码文件的作用。

> 本文按当前仓库实际代码编写。项目主线是：自然语言任务 → AI 解析/目标检测 → RGB 视觉闭环 → J1～J5 运动 → J6 夹取/释放 → 回到 Home。

## 1. 开始前须知

### 1.1 项目包含哪些运行环境

本项目不是只有一个运行环境，而是包含以下几部分：

| 环境                              | 主要用途                          | 是否必须              |
| ------------------------------- | ----------------------------- | ----------------- |
| Windows 10/11 x64 + Python 3.12 | 核心算法、串口控制、单元测试、Windows 操作终端   | 是                 |
| Gemini/OpenNI Windows 环境        | Gemini Pro Plus 彩色、深度、红外和点云实验 | 使用相机时需要           |
| Ubuntu 22.04 独立操作终端             | 在 Ubuntu 上直接通过串口控制机械臂         | 使用 Ubuntu 控制真机时需要 |
| Windows Word/Python 报告环境        | 生成报告、图表及检查 Word 格式            | 仅维护报告时需要          |

### 1.2 默认目录

本文假设仓库位于：

```text
D:\jetarm\project
```

命令通常从父目录 `D:\jetarm` 执行。这一点很重要，因为测试代码使用了 `project.src...` 导入路径。

### 1.3 真机安全要求

- 第一次运行时先使用 `--dry-run` 或不带 `--confirm` 的规划模式。
- 通电前确认舵机 ID、关节方向、机械限位、供电电压和串口号。
- 测试时让机械臂远离人员、桌面边缘及易碰撞物体，并准备随时断电。
- `servo_limit_tester.py` 会让舵机持续运动；没有确认机械结构和方向前，不要添加 `--confirm`。
- J2 明确禁止使用 motor 模式寻找机械限位。

## 2. Windows 主开发环境配置

### 2.1 安装 Python 3.12 和 Git

打开 PowerShell，先检查是否已经安装：

```powershell
py --version
git --version
```

**建议装一下git，不用**

如果命令不存在，可使用 Windows 10/11 自带的 WinGet 安装：

```powershell
winget install --id Python.Python.3.12 -e
winget install --id Git.Git -e
```

安装后关闭并重新打开 PowerShell，再检查：

```powershell
py -3.12 --version
git --version
```

应看到 Python `3.12.x`。WinGet 的官方说明见 [Microsoft Learn](https://learn.microsoft.com/windows/package-manager/winget/)。

### 2.2 获取项目

如果已经存在 `D:\jetarm\project`，只需进入工作区：

（在文件夹目录下右键打开终端）

```powershell
Set-Location D:\jetarm
```

检查当前目录：

```powershell
Get-Location
Test-Path .\project\requirements.txt
```

第二条命令应输出 `True`。

### 2.3 创建虚拟环境

不要把依赖直接安装到系统 Python。执行：

```powershell
Set-Location D:\jetarm
py -3.12 -m venv .venv-jetarm
```

如果 PowerShell 不允许执行激活脚本，只对当前窗口临时放行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

激活虚拟环境：

```powershell
.\.venv-jetarm\Scripts\Activate.ps1
```

确认当前使用的是新环境：

```powershell
python --version
python -c "import sys; print(sys.executable)"
```

第二条命令应指向 `D:\jetarm\.venv-jetarm\Scripts\python.exe`。

以后每次新开 PowerShell，都需要重新执行：

```powershell
Set-Location D:\jetarm
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv-jetarm\Scripts\Activate.ps1
```

退出虚拟环境使用：

```powershell
deactivate
```

### 2.4 安装项目依赖

激活虚拟环境后执行：

```powershell
python -m pip install --upgrade pip
python -m pip install -r .\project\requirements.txt
```

`requirements.txt` 当前包含：

| 包               | 用途                                              |
| --------------- | ----------------------------------------------- |
| `numpy`         | 运动学、矩阵和图像数组计算                                   |
| `opencv-python` | RGB/深度画面显示、颜色检测、图像保存                            |
| `pyserial`      | Windows COM 口和总线舵机通信                            |
| `pygame`        | 点云查看器界面和渲染                                      |
| `pyorbbecsdk2`  | Orbbec/Gemini Python SDK；安装后的导入名为 `pyorbbecsdk` |

验证依赖：

```powershell
python -c "import cv2, numpy, serial, pygame; print('基础依赖安装成功')"
python -c "import pyorbbecsdk; print('Orbbec SDK 导入成功')"
```

如果第二条失败，不影响纯 RGB/OpenNI 路线和机械臂核心代码，可继续参考 3.2 节排查。

### 2.5 运行无硬件验证

先运行全部核心测试：

```powershell
Set-Location D:\jetarm
python -m unittest discover -s project\tests -v
```

再运行几个不会打开串口的命令：

```powershell
python project\src\arm_model.py --position J1=500 --position J2=550 --position J3=550 --position J4=900 --position J5=500 --json
python project\src\arm_model.py --angle-deg J1=0 --angle-deg J2=0 --angle-deg J3=0 --angle-deg J4=0 --angle-deg J5=0 --jacobian --json
python project\src\joint_controller.py --joint J1=520 --current J1=500 --json
python project\src\operation_terminal.py --dry-run
```

说明：

- `arm_model.py` 计算正运动学和 TCP 位置。
- `joint_controller.py` 不带 `--confirm` 时只规划命令，不连接真机。
- `operation_terminal.py --dry-run` 打开模拟界面，不打开串口。

如果窗口无法打开，检查当前 Python 是否包含 Tkinter：

```powershell
python -c "import tkinter; print('Tkinter 可用')"
```

官方 Python 安装器通常已经包含 Tkinter。

## 3. Windows 真机串口与 Gemini 相机配置

### 3.1 总线舵机串口

连接 USB 转串口设备并给机械臂正确供电后，列出串口：

```powershell
python -m serial.tools.list_ports
```

也可以在“设备管理器 → 端口（COM 和 LPT）”中查看，例如 `COM15`。

项目默认波特率为 `115200`。先只读状态：

```powershell
python project\src\bus_servo.py status COM15 1
```

将 `COM15` 和舵机 ID `1` 替换为实际值。读取成功后，再考虑位置控制：

```powershell
python project\src\bus_servo.py servo COM15 1 500 1000
```

上面表示舵机 1 在 1000 ms 内移动到原始位置 500。Motor 模式示例：

```powershell
python project\src\bus_servo.py motor COM15 10 100
python project\src\bus_servo.py motor COM15 10 0
```

第二条是停止命令。对未知舵机不要直接运行 motor 命令。

使用配置和安全校验执行多关节命令：

```powershell
# 只显示计划，不移动
python project\src\joint_controller.py --joint J1=520 --joint J2=560 --com-port COM15 --json

# 确认现场安全后才允许执行
python project\src\joint_controller.py --joint J1=520 --joint J2=560 --com-port COM15 --confirm
```

启动 Windows 真机操作终端：

```powershell
python project\src\operation_terminal.py --com-port COM15
```

### 3.2 Gemini Python SDK 路线

先安装 Microsoft Visual C++ x64 运行库，避免原生扩展缺少 DLL：

```powershell
winget install --id Microsoft.VCRedist.2015+.x64 -e
```

重新打开 PowerShell、激活虚拟环境，然后运行诊断：

```powershell
Set-Location D:\jetarm
.\.venv-jetarm\Scripts\Activate.ps1
python project\src\gemini_windows\diagnose_environment.py
python project\src\gemini_windows\windows_usb_check.py
python project\src\gemini_windows\list_devices.py
```

查看 RGB-D 流：

```powershell
python project\src\gemini_windows\camera_stream_test.py
```

常用按键：`q` 或 `Esc` 退出，`s` 保存当前样本到 `project/data/rgbd_samples/`。

读取相机内参：

```powershell
python project\src\gemini_windows\read_intrinsics.py
```

如果 `pyorbbecsdk` 导入失败，应核对：Python 是否为 x64、版本是否为 3.11～3.13、VC++ 运行库是否安装，以及 wheel 与 Python 版本是否匹配。

### 3.3 Gemini OpenNI + UVC 路线

项目的 OpenNI 包装代码会自动查找工作区下名称以 `gemini` 开头且包含 `windows` 的资料目录，并要求其中存在类似以下路径：

```text
Windows\Windows\OpenNI_v2.3.0.85_..._windows_x64_x86_release
```

当前工作区中的 `gemini深度相机windows资料/` 用于提供这套 SDK。若资料被移动，临时指定路径：

```powershell
$env:GEMINI_OPENNI_ROOT = "D:\你的路径\OpenNI_v2.3.0.85_20220615_1b09bbfd_windows_x64_x86_release"
```

需要永久保存时：

```powershell
[Environment]::SetEnvironmentVariable(
    "GEMINI_OPENNI_ROOT",
    "D:\你的路径\OpenNI_v2.3.0.85_20220615_1b09bbfd_windows_x64_x86_release",
    "User"
)
```

关闭 Orbbec Viewer 等可能占用相机的程序，然后执行：

```powershell
# 查看 OpenCV/UVC 彩色画面
python project\src\gemini_windows\opencv_uvc_color_test.py

# 查看 OpenNI 深度、红外和 UVC 彩色画面
python project\src\gemini_windows\openni_rgbd_viewer.py --single-window

# 查看官方示例列表的入口帮助
python project\src\gemini_windows\run_openni_sample.py --help

# 启动项目组合查看器
python project\src\gemini_windows\run_openni_sample.py viewer

# 连续点云查看
python project\src\gemini_windows\run_openni_sample.py pointcloud-viewer

# 红色物块检测
python project\src\gemini_windows\red_block_detector.py --show-mask
```

如果彩色相机索引不正确，先探测 0～6：

```powershell
python project\src\gemini_windows\opencv_uvc_color_test.py --max-index 6
```

也可以显式指定，例如：

```powershell
python project\src\gemini_windows\opencv_uvc_color_test.py --index 1
```

## 4. Ubuntu 22.04 独立操作终端配置

这一部分不依赖上级 `project/src/`，整个 `ubuntu22_04_operation_terminal/` 可以单独复制到 Ubuntu 22.04。

### 4.1 安装系统依赖

在 Ubuntu 终端执行：

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-tk python3-numpy python3-serial fonts-noto-cjk
```

进入目录并运行安装脚本：

```bash
cd ~/ubuntu22_04_operation_terminal
chmod +x setup.sh run.sh
bash setup.sh
```

脚本会创建 `.venv/`，并通过 `--system-site-packages` 使用 Ubuntu 已安装的 NumPy、Tkinter 和 pyserial，不要求访问 PyPI。

### 4.2 配置串口权限

```bash
sudo usermod -aG dialout "$USER"
```

执行后必须注销并重新登录。重新登录后检查：

```bash
groups
bash run.sh --list-ports
```

### 4.3 运行和测试

先运行无硬件预览：

```bash
bash run.sh --dry-run
```

运行测试：

```bash
.venv/bin/python -m unittest discover -s tests -v
```

连接真机时，可以让程序自动选择唯一串口：

```bash
bash run.sh
```

或明确指定：

```bash
bash run.sh --port /dev/ttyUSB0
```

推荐使用插拔后名称稳定的路径：

```bash
ls -l /dev/serial/by-id/
bash run.sh --port /dev/serial/by-id/usb-你的设备名称
```

配置文件是 `config/terminal.json`。默认 J6 舵机 ID 为 `10`，修改前应先关闭机械臂电源。

## 5. 可选：报告与图表生成环境

报告代码不属于机械臂运行时依赖。建议另建虚拟环境，或在主虚拟环境中额外安装：

```powershell
Set-Location D:\jetarm
.\.venv-jetarm\Scripts\Activate.ps1
python -m pip install Pillow python-docx
```

图表脚本可按文件直接运行，例如：

```powershell
python project\report_build\build_system_architecture_diagram.py
python project\report_build\build_hardware_block_diagram.py
python project\report_build\build_software_flowchart.py
```

PowerShell Word 格式脚本依赖 Windows 桌面版 Microsoft Word 的 COM 接口。例如：

```powershell
powershell -ExecutionPolicy Bypass -File project\report_build\audit_exact_template.ps1 `
  -TemplatePath project\deliverables\中文报告模板.doc `
  -ReportPath project\deliverables\JetArm作品设计报告.docx
```

当前可移植性限制：

- `build_report.py`、`build_template_report.py`、`build_english_report.py` 将 `table_geometry` 的位置写死为开发机上的 `C:\Users\ASUS\.codex\plugins\cache\...`。
- 新电脑仅安装 `Pillow` 和 `python-docx` 后，这三个脚本仍可能报 `No module named table_geometry`。
- 正确修复方式是把 `table_geometry.py` 纳入仓库并改为相对导入，或把路径改成当前机器真实位置；不要复制一个已经失效的绝对版本目录。

## 7. 核心源码功能

### 7.1 `src/` 机械臂核心代码

| 文件                          | 功能                                                                                              | 主要入口/类                                                                         |
| --------------------------- | ----------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| `src/arm_model.py`          | 读取 `joint_servo_map.yaml`，完成原始舵机位置、舵机角、模型角之间的转换；计算 J1～J5 正运动学、TCP 位姿和数值平移雅可比；支持 JSON 命令行输出      | `JetArmKinematicModel`、`forward_kinematics_*()`、`tcp_jacobian_*()`、`main()`    |
| `src/bus_servo.py`          | 实现 `55 55 ID Length Cmd Params Checksum` 半双工串口协议；负责组帧、校验、响应解析、温度/电压/位置读取、Servo 位置模式和 Motor 速度模式 | `BusServoController`、`ServoStatus`、`build_packet()`、`parse_packet()`           |
| `src/joint_controller.py`   | 读取关节配置并把目标位置/角度转换为安全命令；按关节运动档位计算运行时间；J5/J6 可按 motor 模式运行到目标；只有 `--confirm` 才真正连接串口              | `JointCommandPlanner`、`JointCommandExecutor`、`ServoMoveCommand`、`MotorCommand` |
| `src/operation_terminal.py` | Windows Tkinter 图形界面；提供上下、水平摇杆、J5 旋转、J6 开合/抓紧、Home 和全部停止；用雅可比阻尼求解和局部搜索把 TCP 速度转换成 J1～J4 小步位置命令  | `ManualServoRuntime`、`OperationTerminalApp`、`DryRunServoController`            |
| `src/servo_limit_tester.py` | 按指定方向和速度驱动单个舵机，持续读取位置；当位置在容差内稳定指定时间时判断到达限位/接触；超时或异常时停止，并可恢复 Servo 模式                            | `ServoLimitTester`、`PositionStabilityTracker`、`LimitTestResult`                |

核心调用关系：

```text
joint_servo_map.yaml
    ├─> arm_model.py ───────────────> TCP 位姿/雅可比
    ├─> joint_controller.py ────────> 安全关节命令
    └─> operation_terminal.py ──────> 人机操作与笛卡尔小步控制

joint_controller.py / operation_terminal.py / servo_limit_tester.py
    └─> bus_servo.py ───────────────> COM 口 ─> 总线舵机
```

### 7.2 `src/gemini_windows/` 相机代码

| 文件                         | 功能                                                                           |
| -------------------------- | ---------------------------------------------------------------------------- |
| `__init__.py`              | 将目录标记为 Python 包；本身不包含业务逻辑                                                    |
| `gemini_common.py`         | pyorbbecsdk 通用工具：安全导入 SDK、选择 RGB-D 流、把彩色帧转 BGR、把深度转毫米、深度着色、局部中值深度、像素反投影和读取内参 |
| `diagnose_environment.py`  | 输出 Python、平台和工作目录；检查 NumPy、OpenCV、pyorbbecsdk，并在子进程中验证原生 SDK 是否会导入失败         |
| `list_devices.py`          | 通过 pyorbbecsdk 枚举已连接的 Orbbec/Gemini 设备                                       |
| `camera_stream_test.py`    | pyorbbecsdk RGB-D 实时测试；点击像素显示深度/三维坐标，按 `s` 保存样本                              |
| `read_intrinsics.py`       | 读取彩色和深度相机内参并以 JSON 输出                                                        |
| `opencv_uvc_color_test.py` | 用 OpenCV 探测 Windows UVC 摄像头索引，显示 Gemini 彩色画面并保存图片                            |
| `openni2_ctypes.py`        | 使用 `ctypes` 直接包装捆绑的 OpenNI2 C API；查找 DLL、枚举模式、配置激光/LDP、创建数据流和读取帧             |
| `openni_rgbd_viewer.py`    | 组合 OpenNI 深度/红外与 OpenCV UVC 彩色流；支持独立/合并窗口、镜像、曝光、激光控制、模式列表和快照保存               |

## 14. 常见问题

### 14.1 `python` 或 `py` 不是命令

重新安装 Python 3.12，并在安装后关闭、重开 PowerShell：

```powershell
winget install --id Python.Python.3.12 -e
py -3.12 --version
```

### 14.2 PowerShell 禁止运行 `Activate.ps1`

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv-jetarm\Scripts\Activate.ps1
```

这只影响当前 PowerShell 窗口。

### 14.3 测试提示 `No module named project`

通常是运行目录错误。回到父目录：

```powershell
Set-Location D:\jetarm
python -m unittest discover -s project\tests -v
```

### 14.4 串口打不开或权限不足

Windows：

```powershell
python -m serial.tools.list_ports
```

确认串口没有被其他软件占用。Ubuntu：

```bash
sudo usermod -aG dialout "$USER"
```

然后注销并重新登录。

### 14.5 Gemini 有彩色画面但没有深度

- 彩色通常走 UVC/OpenCV，深度和红外走 OpenNI/SDK，两者是不同链路。
- 关闭 Orbbec Viewer 和其他占用相机的程序。
- 运行 `windows_usb_check.py` 和 `diagnose_environment.py`。
- 检查 `GEMINI_OPENNI_ROOT` 是否指向包含 `samples/bin/OpenNI2.dll` 的 OpenNI 根目录。

### 14.6 报告脚本提示找不到 `table_geometry`

这是当前代码中的绝对路径依赖，不是 pip 包缺失。参见第 5 节“当前可移植性限制”。

## 15. 推荐的初学者学习顺序

1. 完成第 2 节，只运行单元测试和 `--dry-run`。
2. 阅读 `config/joint_servo_map.yaml`，理解舵机原始位置、模型角和关节方向。
3. 阅读并运行 `arm_model.py`，观察不同关节角对 TCP 的影响。
4. 阅读 `bus_servo.py` 和对应测试，理解协议帧，但暂不移动真机。
5. 连接硬件后只读舵机状态，再使用很小的位置变化验证单个关节。
6. 使用 Gemini 代码分别验证彩色和深度链路。
7. 最后再使用操作终端、视觉闭环和多关节真机控制。
