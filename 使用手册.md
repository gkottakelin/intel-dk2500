# JetArm 项目使用手册

本文档说明 `project/` 开发前需要准备什么、当前代码分别负责什么、以及如何通过命令行或 Python 代码调用。当前项目主要包含两条已经打通的链路：

- Gemini Pro Plus 深度相机：Windows 阶段通过 OpenNI/UVC/OpenCV 读取 RGB-D、点云和红色目标检测。
- 总线舵机/机械臂底层控制：通过 Python 串口协议读状态、发位置模式和 motor 模式命令。

ROS 2/MoveIt 2 目前属于下一阶段运动规划接入环境，当前 `project/src` 下还没有正式 ROS 2 package。

## 1. 开发前准备工作

### 1.1 推荐开发环境

当前建议保留两套环境：

| 环境 | 用途 |
|---|---|
| Windows + Python 虚拟环境 | 运行 Gemini 相机调试、OpenNI/UVC、舵机串口控制代码 |
| WSL2 Ubuntu 22.04 + ROS 2 Humble | 后续六轴机械臂运动规划、MoveIt 2、ROS 2 节点开发 |

Windows 项目目录：

```text
D:\jetarm\project
```

在 WSL Ubuntu 中访问同一目录：

```bash
/mnt/d/jetarm/project
```

注意：ROS 2 工作区建议放在 Ubuntu 自己的 Linux 文件系统中，例如：

```bash
~/ros2_ws
~/ros2_ws/src
```

不要长期在 `/mnt/d/jetarm/project` 里编译 ROS 2 包，速度和软链接兼容性都可能较差。

### 1.2 Windows Python 环境

推荐在 `D:\jetarm` 下使用已有或新建的虚拟环境 `.venv-gemini`。

创建虚拟环境：

```powershell
cd D:\jetarm
python -m venv .venv-gemini
```

激活虚拟环境：

```powershell
D:\jetarm\.venv-gemini\Scripts\activate
```

安装项目依赖：

```powershell
pip install -r D:\jetarm\project\requirements.txt
```

主要依赖：

| 依赖 | 用途 |
|---|---|
| `numpy` | 图像、深度图、点云数组处理 |
| `opencv-python` | UVC 彩色图读取、图像显示、红色目标检测 |
| `pyserial` | 总线舵机串口通信 |
| `pygame` | 点云可视化窗口 |
| `pyorbbecsdk2` | Orbbec SDK 诊断/备选 RGB-D 读取路径 |

### 1.3 Gemini/OpenNI 资料准备

当前相机主链路依赖厂商 OpenNI Windows 资料包。代码会自动在 `D:\jetarm` 下寻找名称类似下面的目录：

```text
OpenNI_v2.3.0.85_20220615_1b09bbfd_windows_x64_x86_release
```

如果资料目录移动过，可以手动设置环境变量 `GEMINI_OPENNI_ROOT` 指向 OpenNI 根目录：

```powershell
$env:GEMINI_OPENNI_ROOT="D:\path\to\OpenNI_v2.3.0.85_20220615_1b09bbfd_windows_x64_x86_release"
```

运行相机代码前请确认：

- Gemini Pro Plus 已连接 USB。
- Orbbec Viewer 已关闭，避免占用设备。
- Windows 设备管理器能看到相机/UVC 设备。
- 使用 OpenNI 例程前，确保厂商资料包完整。

### 1.4 总线舵机准备

运行舵机控制代码前请确认：

- USB 转串口驱动已安装，例如 CH340/CH341。
- 能在 Windows 设备管理器中看到 COM 口，例如 `COM3`。
- 总线舵机供电正常，地线和通信线连接正确。
- 默认波特率为 `115200`。
- 目标位置范围通常为 `0..1000`，对应约 `0..240` 度。

### 1.5 ROS 2 准备

如果进入运动规划阶段，在 Ubuntu 22.04 中使用 ROS 2 Humble。

每次打开 Ubuntu 后，如果你不想自动加载 ROS 2，就手动执行：

```bash
source /opt/ros/humble/setup.bash
```

进入工作区：

```bash
cd ~/ros2_ws
```

如果工作区已经编译过：

```bash
source install/setup.bash
```

如果工作区还没有包，`colcon build` 出现下面结果是正常的：

```text
Summary: 0 packages finished
```

这表示环境可用，只是 `src` 下还没有 ROS 2 package。

## 3. 当前代码作用和调用方式

下面命令默认从 Windows PowerShell 的 `D:\jetarm` 目录运行：

```powershell
cd D:\jetarm
D:\jetarm\.venv-gemini\Scripts\activate
```

如果你已经在 `D:\jetarm\project` 目录中，则命令里的 `project/` 前缀可以省略，例如 `python src\bus_servo.py ...`。

### 3.1 总线舵机控制：`src/bus_servo.py`

作用：

- 按总线舵机协议生成/解析通信帧。
- 读取单个舵机温度、当前位置、电压。
- 发送位置模式命令。
- 发送 motor 连续旋转模式命令。
- 提供命令行和 Python API 两种调用方式。

协议要点：

| 项目 | 内容 |
|---|---|
| 帧头 | `55 55` |
| 帧格式 | `55 55 ID Length Cmd Params... Checksum` |
| 默认波特率 | `115200` |
| 单个舵机 ID | `0..253` |
| 广播 ID | `254`，仅用于写入命令 |
| 位置范围 | `0..1000` |
| 运行时间 | `0..30000 ms` |
| motor 占空比速度 | `-1000..1000` |
| 固定速度模式 | `-50..50` |

命令行调用：

```powershell
python project\src\bus_servo.py status COM3 1
```

读取 `COM3` 上 ID 为 `1` 的舵机状态。

```powershell
python project\src\bus_servo.py servo COM3 1 500 1000
```

让 ID 为 `1` 的舵机移动到位置 `500`，运行时间 `1000 ms`。

```powershell
python project\src\bus_servo.py motor COM3 1 100
```

让 ID 为 `1` 的舵机进入 motor 模式，占空比速度 `100`。

```powershell
python project\src\bus_servo.py motor COM3 1 20 --fixed-speed-mode
```

使用固定速度模式，速度范围为 `-50..50`。

Python 调用：

```python
from project.src.bus_servo import BusServoController

with BusServoController("COM3") as servo:
    status = servo.read_status(1)
    print(status.temperature_c, status.position, status.voltage_mv)

    servo.move_servo(1, target_position=500, run_time_ms=1000)
    servo.set_motor_speed(1, speed=100)
```

便捷函数调用：

```python
from project.src.bus_servo import read_servo_status, set_servo_position, set_motor_speed

status = read_servo_status("COM3", 1)
set_servo_position("COM3", 1, target_position=500, run_time_ms=1000)
set_motor_speed("COM3", 1, speed=100)
```

```powershell
python project\src\gemini_windows\run_openni_sample.py simple-viewer
```

```text

```

含义是 UVC 彩色路径 + 镜像修正。

```powershell
python project\src\gemini_windows\run_openni_sample.py viewer
```

运行项目自写的 OpenNI 深度/红外 + UVC 彩色 viewer。

```powershell
python project\src\gemini_windows\run_openni_sample.py depth-viewer
```

当前等价于 `viewer`。

```powershell
python project\src\gemini_windows\run_openni_sample.py pointcloud
```

运行厂商 `GeneratePointCloud.exe`，生成 `.ply` 点云文件。该示例默认 50 帧后停止。

### 3.4 OpenNI/UVC RGB-D 查看器：`src/gemini_windows/openni_rgbd_viewer.py`

作用：

- 通过 OpenNI 读取深度流。
- 可选读取红外流。
- 通过 OpenCV/UVC 读取彩色流。
- 显示深度、红外、彩色窗口。
- 可保存彩色图、深度预览图、深度 `.npy`、红外数据。
- 可控制激光、近距离保护、镜像等参数。

直接运行：

```powershell
python project\src\gemini_windows\openni_rgbd_viewer.py
```

单窗口显示：

```powershell
python project\src\gemini_windows\openni_rgbd_viewer.py --single-window
```

列出支持的视频模式：

```powershell
python project\src\gemini_windows\openni_rgbd_viewer.py --list-modes
```

指定彩色摄像头 index：

```powershell
python project\src\gemini_windows\openni_rgbd_viewer.py --color-index 0
```

只看深度，不开彩色：

```powershell
python project\src\gemini_windows\openni_rgbd_viewer.py --no-color
```

关闭红外：

```powershell
python project\src\gemini_windows\openni_rgbd_viewer.py --no-ir
```

保存数据：

- 按 `s` 保存样本到 `project/data/rgbd_samples`。
- 按 `q` 或 `ESC` 退出。

常用参数：

| 参数 | 作用 |
|---|---|
| `--depth-width` / `--depth-height` | 深度图期望分辨率，默认 `640x400` |
| `--color-width` / `--color-height` | 彩色图期望分辨率，默认 `640x480` |
| `--fps` | 帧率，默认 `30` |
| `--min-depth-mm` / `--max-depth-mm` | 深度显示范围 |
| `--mirror-color` | 彩色图水平翻转 |
| `--mirror-depth` | 深度图水平翻转 |
| `--laser-off` | 启动时不打开激光 |
| `--near-protection-on` | 打开近距离保护 |

### 3.5 OpenNI2 C API 封装：`src/gemini_windows/openni2_ctypes.py`

作用：

- 用 `ctypes` 包装 OpenNI2 C API。
- 自动加载 `OpenNI2.dll` 和驱动目录。
- 打开 Gemini 设备。
- 创建深度/红外流。
- 读取帧数据为 `numpy.ndarray`。
- 设置视频模式、镜像、激光、LDP/近距离保护。

通常不直接命令行运行，而是被以下模块调用：

- `openni_rgbd_viewer.py`
- `red_block_detector.py`
- `run_openni_sample.py`

Python 调用示例：

```python
from project.src.gemini_windows.openni2_ctypes import (
    OpenNI2,
    ONI_SENSOR_DEPTH,
    ONI_PIXEL_FORMAT_DEPTH_1_MM,
    ONI_PIXEL_FORMAT_DEPTH_100_UM,
)

with OpenNI2() as device:
    depth = device.create_stream(
        ONI_SENSOR_DEPTH,
        width=640,
        height=400,
        fps=30,
        pixel_formats=[ONI_PIXEL_FORMAT_DEPTH_1_MM, ONI_PIXEL_FORMAT_DEPTH_100_UM],
    )
    depth.start()
    frame = depth.read_frame()
    print(frame.data.shape, frame.frame_index)
```

### 3.8 红色方块检测：`src/gemini_windows/red_block_detector.py`

作用：

- 使用 UVC 彩色图检测红色目标。
- 使用 OpenNI 深度图匹配目标区域深度。
- 输出目标彩色中心、深度像素、深度值、相机坐标系下的 `XYZ`。
- 可保存彩色图、深度图、mask 和 JSON 检测结果。

运行：

```powershell
python project\src\gemini_windows\red_block_detector.py
```

显示 mask：

```powershell
python project\src\gemini_windows\red_block_detector.py --show-mask
```

指定彩色摄像头：

```powershell
python project\src\gemini_windows\red_block_detector.py --color-index 0
```

调整红色目标最小面积：

```powershell
python project\src\gemini_windows\red_block_detector.py --min-area 800
```

调整深度有效范围：

```powershell
python project\src\gemini_windows\red_block_detector.py --min-depth-mm 100 --max-depth-mm 4000
```

如果彩色图和深度图存在小偏移，可用像素偏移修正：

```powershell

```

窗口操作：

| 操作 | 作用 |
|---|---|
| `s` | 保存调试帧和 JSON |
| `q` / `ESC` | 退出 |

控制台输出示例结构：

```json
{
  "color_center": [320, 240],
  "depth_pixel": [318, 200],
  "depth_mm": 536.0,
  "camera_xyz_mm": [12.3, -8.1, 536.0],
```

后续机械臂抓取时，`camera_xyz_mm` 还需要通过手眼标定转换为机械臂基坐标系下的目标位置。

## 4. 推荐调试顺序

### 4.1 相机链路

第一步，检查 Windows 是否看到设备：

```powershell
python project\src\gemini_windows\windows_usb_check.py
```

第二步，检查 Python 依赖：

```powershell
python project\src\gemini_windows\diagnose_environment.py
```

第三步，测试 UVC 彩色图：


## 6. 后续 ROS 2/运动规划接入建议

当前代码还不是 ROS 2 package。后续建议在 Ubuntu 22.04 的 `~/ros2_ws/src` 下逐步建立：

| ROS 2 包 | 作用 |
|---|---|
| `jetarm_description` | 机械臂 URDF/Xacro、mesh、关节限位、TCP |
| `jetarm_moveit_config` | MoveIt 2 运动规划配置 |
| `jetarm_control` | ros2_control 硬件接口或桥接当前 `bus_servo.py` |
| `jetarm_vision` | 相机目标检测、坐标转换、手眼标定结果发布 |
| `jetarm_bringup` | launch 总入口 |

迁移思路：

1. 保留 `bus_servo.py` 作为底层协议层。
2. 在 `jetarm_control` 中封装 ROS 2 节点或 `ros2_control` hardware interface。
3. 将 `red_block_detector.py` 的输出改为 ROS 2 topic，例如发布目标在相机坐标系下的位置。
4. 通过手眼标定得到 `camera_frame -> base_link` 变换。
5. MoveIt 2 接收目标位姿，规划六轴机械臂轨迹。
6. 控制层将规划轨迹转换为各关节舵机位置命令。

打开 Ubuntu 后的 ROS 2 手动加载流程：

```bash
source /opt/ros/humble/setup.bash
cd ~/ros2_ws
source install/setup.bash
```

如果新增或修改 ROS 2 包：

```bash
source /opt/ros/humble/setup.bash
cd ~/ros2_ws
colcon build
source install/setup.bash
```

## 7. 常见问题

### 7.1 `ros2: command not found`

说明当前 Ubuntu 终端没有加载 ROS 2：

```bash
source /opt/ros/humble/setup.bash
```

### 7.2 `colcon: command not found`

说明没有安装 colcon，或没有加载环境。先加载 ROS 2：

```bash
source /opt/ros/humble/setup.bash
```

如果仍然没有：

```bash
sudo apt install -y python3-colcon-common-extensions
```

### 7.3 `Summary: 0 packages finished`

这是正常情况，表示当前 ROS 2 工作区没有 package。

### 7.4 相机无法打开

按顺序检查：

1. 关闭 Orbbec Viewer。
2. 重新插拔 USB。
3. 运行 `windows_usb_check.py`。
4. 运行 `opencv_uvc_color_test.py` 检查彩色 UVC。
5. 运行 `run_openni_sample.py depth` 检查深度 OpenNI。
6. 检查 `GEMINI_OPENNI_ROOT` 是否指向正确 OpenNI 目录。

### 7.5 舵机没有响应

按顺序检查：

1. COM 口是否正确。
2. 舵机供电是否正常。
3. 波特率是否为 `115200`。
4. 舵机 ID 是否正确。
5. 总线接线是否正确。
6. 先运行 `status`，再运行位置或 motor 命令。

## 8. 当前阶段最重要的下一步

在运动规划前，建议先补齐以下数据：

1. 六轴机械臂每个关节的零位、方向、角度范围和舵机 ID 对应关系。
2. 舵机位置值 `0..1000` 与真实关节角度的映射关系。
3. 机械臂连杆长度、关节轴方向、TCP 坐标。
4. 相机固定位置和手眼标定结果。
5. 机械臂基坐标系、相机坐标系、末端工具坐标系定义。

这些数据齐全后，才能稳定进入 URDF 建模、MoveIt 2 配置、逆运动学、轨迹规划和真实机械臂执行。
