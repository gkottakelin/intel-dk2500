# 项目方法总结

> 本文总结当前 RGB 视觉闭环路线中已经使用或确定采用的方法。

## 1. 项目工程化方法

| 使用的方法 | 解决的问题 | 实现的功能 |
|---|---|---|
| 项目目录分层管理 | 原始资料、代码、规划文档混在一起，不利于持续开发 | 建立 `project/` 作为开发目录，集中管理文档、代码、配置、测试 |
| 当前主线索引法 | 文档入口分散，容易执行过时路线 | 建立 `docs/README.md` 和根 `README.md`，只保留当前 RGB 视觉闭环路线 |
| 模块化架构法 | 相机、AI、规划、舵机之间直接耦合会导致调试困难 | 将系统拆分为视觉模块、语言理解模块、运动规划模块、执行控制模块 |
| 阶段化开发法 | 机械臂系统同时包含视觉、AI、规划、舵机，复杂度高 | 将系统拆成 RGB 检测、视觉闭环标定、抓取闭环、放置闭环等阶段 |

## 2. 总线舵机控制方法

| 使用的方法 | 解决的问题 | 实现的功能 |
|---|---|---|
| UART 串口通信方法 | 上位机需要通过串口与舵机控制板通信 | 使用 COM 口和 `115200bps` 串口速率发送舵机指令 |
| 协议帧组包方法 | 舵机通信协议要求固定帧头、ID、长度、指令、参数和校验 | 在 Python 中生成符合手册要求的舵机控制帧 |
| 校验和校验方法 | 防止舵机接收错误帧或解析错误数据 | 实现协议校验和计算与返回帧校验 |
| 状态读取解析方法 | 需要读取舵机温度、角度、电压来判断运行状态 | 实现 `read_servo_status(com_port, servo_id)` |
| Servo 位置控制方法 | J1-J5 需要运动到指定位置 | 实现 `set_servo_position(com_port, servo_id, target_position, run_time_ms)` |
| Motor 速度控制方法 | J6 夹爪需要按速度夹取和释放 | 实现 `set_motor_speed(com_port, servo_id, speed)` |
| 假串口单元测试方法 | 没有硬件时无法稳定测试协议代码 | 使用测试代码验证组帧、校验、状态解析等逻辑 |

## 3. RGB 视觉闭环方法

| 使用的方法 | 解决的问题 | 实现的功能 |
|---|---|---|
| RGB-only 感知方法 | 当前只允许读取彩色图 | 用 RGB 图完成目标检测、夹爪定位和闭环修正 |
| 眼在手上相机方法 | 相机安装在 J5/J6 附近，会随 J5 转动 | 让相机视角跟随末端运动，并直接观察夹爪和目标 |
| 夹爪颜色标记方法 | 黑色夹爪本体在图像中难以稳定分割 | 使用橙色爪尖作为主定位特征，紫红色区域作为辅助特征 |
| 夹爪中心线估计方法 | 需要把目标夹在两爪中间 | 根据两侧橙色爪尖估计夹爪端点、中心点和中心线 |
| 图像误差闭环方法 | 实际舵机有误差，开环规划会偏 | 用目标像素位置和夹爪中心线误差不断修正 J1-J5 |
| 视觉伺服雅可比标定方法 | 不确定每个关节小步动作对图像误差的影响 | 通过实测建立像素误差到关节增量的局部响应关系 |

## 4. AI 自然语言与任务规划方法

| 使用的方法 | 解决的问题 | 实现的功能 |
|---|---|---|
| 自然语言结构化方法 | 人类输入是自然语言，机械臂需要明确动作参数 | 将“抓取红色物块”转换为目标类别、目标属性、动作类型 |
| 结构化 JSON 方法 | 云端大模型输出不能直接执行自然语言 | 规定 `task_type、target、destination、steps、constraints` 等字段 |
| 任务分解方法 | 一个自然语言命令可能包含识别、移动、抓取、放置多个步骤 | 把任务拆解为感知、定位、规划、执行、确认 |
| 安全限位方法 | 机械臂可能撞限位、撞桌面或自碰撞 | 使用现有关节限位、桌面高度 `z=0` 和任务后回 home 约束 |
| 执行闭环方法 | 舵机执行后需要确认状态 | 通过状态读取接口检查温度、电压、角度/J6 位置变化，判断动作是否完成 |

## 5. ROS2 机械臂描述和仿真方法

| 使用的方法 | 解决的问题 | 实现的功能 |
|---|---|---|
| URDF/Xacro 建模方法 | 需要在 ROS2/RViz 中看到机械臂结构和关节 | 建立 `jetarm_description` 描述包 |
| GUI 原始值映射方法 | `joint_state_publisher_gui` 显示的是关节角，不符合 J6 原始位置范围 | 使用 `joint_state_mapper.py` 将 J6 GUI 原始值 `0..1000` 映射到显示角度 |
| mimic 夹爪方法 | J6 只有一个舵机参数，但视觉上两根爪子都要运动 | 使用一个主动 `joint6_gripper` 和一个 mimic 关节显示剪刀状双爪 |
| ROS2 参数字符串包装方法 | Humble launch 中 xacro 命令输出会被 YAML 参数解析误判 | 使用 `ParameterValue(..., value_type=str)` 包装 `robot_description` |
