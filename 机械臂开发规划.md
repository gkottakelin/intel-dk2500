# JetArm 六轴机械臂开发规划

本文档只描述当前 RGB 视觉闭环开发路线。

## 1. 当前执行路线

```text
RGB 彩色图
-> 云端大模型任务解析和目标检测
-> 夹爪/目标像素级视觉闭环
-> J1-J5 servo 模式运动
-> J6 motor 模式夹取
```

已确认条件：

| 项目 | 当前约定 |
|---|---|
| home | `J1=500, J2=550, J3=550, J4=900, J5=500, J6=360` |
| 相机 | 安装在 J5/J6 附近，会随 J5 转动，并能看到夹爪 |
| 视觉输入 | 只读取 RGB 彩色图像 |
| 目标 | 抓取目标都在桌面上，桌面高度定义为 `z=0` |
| 夹爪标记 | 橙色爪尖作为主定位特征，紫红色区域作为辅助特征 |
| J1-J5 | servo 模式 |
| J6 | motor 模式 |
| 任务结束 | 每次完成后回 home |

完整方案见 `docs/视觉闭环运动规划开发方案.md`。

## 2. 第一版最小闭环

```text
回 home
-> 打开夹爪到 J6=360
-> 读取 RGB 彩色图
-> 检测目标和夹爪橙色爪尖
-> 用视觉误差迭代调整 J1-J5
-> 让目标落在两爪中心线上
-> J6 motor 模式夹取
-> 抬起
-> 移动到目标区域
-> 视觉确认后释放
-> 回 home
```

成功标准：

- 能稳定识别桌面上的目标物体。
- 能稳定识别夹爪橙色末端标记。
- 能通过闭环把目标调整到两爪中间。
- 夹取时 J6 以速度 `100` 运动，连续 `2s` 位置不变后认为夹到物体，再提高到 `300` 夹紧。
- 释放和任务完成后能回 home。
- 失败时能停止在安全姿态，不继续乱动。

## 3. 开发原则

1. 先标定，再规划。

   没有关节零位、舵机映射、夹爪末端像素特征、相机内参和 TCP 偏移，闭环会反复震荡。

2. 先仿真，再真机。

   关节方向、限位、URDF、J6 夹爪显示和 launch 参数先在 RViz 中验证。

3. 先单关节，再多关节。

   每个舵机先单独低速测试方向、范围、映射，再组合执行。

4. 先图像闭环，再复杂任务。

   第一版先把目标稳定夹在两爪中间，再做放置、叠放和自然语言组合任务。

5. AI 只做任务理解，不直接控制舵机。

   云端大模型输出必须先变成结构化 JSON，再经过本地安全检查、规划和执行。

## 4. 模块结构

```text
自然语言输入
  -> 云端大模型任务解析
  -> 结构化任务 JSON

RGB 视觉感知
  -> 目标检测
  -> 夹爪橙色爪尖检测
  -> 目标/夹爪像素误差

视觉闭环规划
  -> 像素误差
  -> 视觉伺服雅可比
  -> J1-J5 增量命令

执行控制
  -> J1-J5 servo 位置命令
  -> J6 motor 速度命令
  -> 状态读取
  -> 安全停止
```

## 5. ROS2 包规划

| ROS2 包 | 作用 |
|---|---|
| `jetarm_description` | 机械臂 URDF/Xacro、关节限位、TCP、J6 夹爪显示 |
| `jetarm_control` | 当前 `bus_servo.py` 的 ROS2 封装、关节状态、命令执行 |
| `jetarm_vision` | RGB 相机数据接入、目标检测、夹爪标记检测 |
| `jetarm_calibration` | 相机内参、夹爪像素标定、视觉伺服雅可比标定 |
| `jetarm_task` | 抓取任务流程、自然语言任务解析后的结构化执行 |
| `jetarm_bringup` | 一键启动 launch、设备参数加载 |

## 6. 云端大模型输出格式

云端大模型只输出结构化任务，不输出舵机命令。

```json
{
  "task_id": "task_001",
  "task_type": "pick_place",
  "target": {
    "description": "red block",
    "color": "red",
    "object_type": "block"
  },
  "destination": {
    "description": "blue box",
    "color": "blue",
    "object_type": "box"
  },
  "steps": [
    {"action": "observe"},
    {"action": "align_gripper_to_target"},
    {"action": "grasp"},
    {"action": "align_gripper_to_destination"},
    {"action": "release"},
    {"action": "return_home"}
  ],
  "constraints": {
    "table_z": 0,
    "return_home_after_task": true
  }
}
```

本地程序必须检查：

- `task_type` 是否在白名单内。
- 目标是否能在 RGB 图中检测到。
- 目标和放置位置是否在允许工作区内。
- J1-J5 目标位置是否在限位内。
- J6 是否按 motor 模式执行。
