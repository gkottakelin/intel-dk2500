# JetArm 六轴机械臂开发规划

本文档面向下一阶段机械臂运动规划、ROS 2 接入、视觉定位和真实抓取闭环开发。当前项目已经具备两条基础能力：

- Gemini Pro Plus 相机数据链路已打通，可在 Windows 阶段读取彩色、深度、点云，并完成红色目标检测。
- 总线舵机控制代码已完成基础封装，可通过 Python 串口接口读取舵机状态、发送位置和速度命令。

下一阶段的核心目标是：把“目标在相机中的三维位置”稳定转换成“六轴机械臂可执行的安全关节轨迹”，最终完成视觉抓取和放置任务。

## 1. 总体目标

### 1.1 第一版最小闭环

第一版不要一开始追求自然语言、复杂避障和全自动抓取。先完成一个可验证的最小闭环：

```text
识别红色方块
-> 获取相机坐标系下的目标三维坐标
-> 转换到机械臂基坐标系
-> 生成预抓取点和抓取点
-> 求解六轴关节角
-> 下发舵机位置命令
-> 抓取
-> 抬起
-> 放到固定区域
```

第一版成功标准：

- 能稳定识别桌面上的单个红色方块。
- 能输出目标相机坐标和机械臂基坐标。
- 机械臂能移动到目标上方，而不是直接碰撞目标。
- 抓取动作低速、安全、可重复。
- 失败时能停在安全姿态，不继续乱动。

### 1.2 第二版增强目标

在第一版稳定后，再增强：

- 支持多颜色/多目标选择。
- 支持 MoveIt 2 碰撞检测和轨迹规划。
- 支持 RViz 可视化。
- 支持自然语言任务，例如“把红色方块放到左边”。
- 支持 DK2500 上部署 ROS 2 全链路。

## 2. 开发原则

1. 先标定，再规划。
   
   没有关节零位、舵机映射、相机外参和 TCP 偏移，运动规划一定会偏。

2. 先仿真，再真机。
   
   所有关节方向、限位、URDF、IK 和轨迹都先在 RViz/仿真里验证。

3. 先单关节，再多关节。
   
   每个舵机先单独低速测试方向、范围、映射，再组合执行。

4. 先固定点抓取，再视觉抓取。
   
   先让机械臂能去固定空间点，再接入相机目标点。

5. AI 只做任务理解，不直接控制舵机。
   
   自然语言输出必须先变成结构化任务，再经过安全检查、规划和执行。

## 3. 推荐技术路线

### 3.1 开发环境分工

| 环境 | 作用 | 推荐路径 |
|---|---|---|
| Windows + `.venv-gemini` | 相机 OpenNI/UVC 调试、舵机串口调试 | `D:\jetarm\project` |
| WSL2 Ubuntu 22.04 | ROS 2 Humble、MoveIt 2、URDF、仿真和规划 | `~/ros2_ws` |
| 后续 DK2500 Ubuntu 22.04 | 真实部署运行 | `~/ros2_ws` |

注意：

- Windows 项目在 WSL 里是 `/mnt/d/jetarm/project`。
- ROS 2 包建议放在 Ubuntu 内部 `~/ros2_ws/src`，不要长期在 `/mnt/d` 下编译。
- 用户当前不希望自动加载 ROS 2，因此每次打开 Ubuntu 后手动执行：

```bash
source /opt/ros/humble/setup.bash
cd ~/ros2_ws
source install/setup.bash
```

### 3.2 是否使用 ROS 2/MoveIt 2

建议使用 ROS 2 + MoveIt 2，但采用分阶段接入。

第一阶段可以不依赖 MoveIt 2，只做：

- 舵机映射
- 关节限位
- 正逆运动学
- 点到点关节插补
- 安全检查

当机械臂模型和基础动作稳定后，引入 MoveIt 2：

- URDF/Xacro 建模
- SRDF 规划组配置
- 碰撞检测
- 轨迹规划
- RViz 交互调试
- 后续扩展到复杂场景避障

## 4. 系统架构

### 4.1 模块结构

```text
视觉感知
  -> 目标检测
  -> 深度测距
  -> 相机坐标

坐标转换
  -> 相机内参
  -> 手眼标定
  -> base_link 坐标

运动规划
  -> 目标位姿
  -> IK 求解
  -> 轨迹生成
  -> 碰撞/限位检查

执行控制
  -> 关节角转舵机位置
  -> 串口下发
  -> 状态读取
  -> 安全停止
```

### 4.2 ROS 2 包规划

后续在 `~/ros2_ws/src` 下建议建立：

| ROS 2 包 | 作用 |
|---|---|
| `jetarm_description` | 机械臂 URDF/Xacro、mesh、关节限位、TCP、相机安装位姿 |
| `jetarm_moveit_config` | MoveIt 2 配置、规划组、碰撞矩阵、RViz 配置 |
| `jetarm_control` | 当前 `bus_servo.py` 的 ROS 2 封装、关节状态、轨迹执行 |
| `jetarm_vision` | 相机数据接入、目标检测、深度定位 |
| `jetarm_calibration` | 相机内参、手眼标定、桌面平面标定 |
| `jetarm_task` | 抓取任务流程、自然语言任务解析后的结构化执行 |
| `jetarm_bringup` | 一键启动 launch、设备参数加载 |

建议目录：

```text
~/ros2_ws/src/
  jetarm_description/
  jetarm_moveit_config/
  jetarm_control/
  jetarm_vision/
  jetarm_calibration/
  jetarm_task/
  jetarm_bringup/
```

## 5. 必须先补齐的机械臂数据

这是下一阶段最关键的表。没有这些数据，不要直接做复杂规划。

### 5.1 舵机和关节映射表

建议在 `project/config/joint_servo_map.yaml` 中维护：

```yaml
joints:
  joint1:
    servo_id: 1
    name: base_yaw
    direction: 1
    zero_position: 500
    min_position: 150
    max_position: 850
    min_angle_deg: -90
    max_angle_deg: 90
  joint2:
    servo_id: 2
    name: shoulder_pitch
    direction: -1
    zero_position: 500
    min_position: 180
    max_position: 820
    min_angle_deg: -75
    max_angle_deg: 75
```

每个关节都要确认：

- 舵机 ID
- 对应机械臂第几轴
- 正方向
- 零位位置值
- 安全最小/最大舵机位置
- 位置值与角度的转换关系
- 是否有机械硬限位

### 5.2 机械尺寸表

建议记录：

| 数据 | 说明 |
|---|---|
| `base_height` | 基座高度 |
| `link1_length` | 肩关节到肘关节距离 |
| `link2_length` | 肘关节到腕关节距离 |
| `wrist_length` | 腕部到末端 TCP 距离 |
| `gripper_offset` | 夹爪中心相对末端法兰偏移 |
| `camera_offset` | 相机相对机械臂基座或末端的位姿 |

### 5.3 坐标系定义

必须统一坐标系，否则后续很难调。

建议定义：

| 坐标系 | 作用 |
|---|---|
| `base_link` | 机械臂基座坐标系 |
| `joint1_link` 到 `joint6_link` | 各关节连杆坐标系 |
| `tool0` | 第六轴末端法兰坐标系 |
| `tcp` | 实际夹爪抓取中心 |
| `camera_color_optical_frame` | 彩色相机光学坐标系 |
| `camera_depth_optical_frame` | 深度相机光学坐标系 |
| `object_frame` | 被抓取目标坐标系 |

第一版如果相机固定在桌面/外部，使用“眼在手外”标定：

```text
camera_frame -> base_link
```

如果相机装在末端，使用“眼在手上”标定：

```text
camera_frame -> tool0
tool0 -> base_link
```

## 6. 分阶段开发计划

### 阶段 0：硬件和环境冻结

目标：让后续调试不再反复被环境问题打断。

任务：

1. 固定开发电脑和 WSL2 Ubuntu 22.04 环境。
2. 确认 ROS 2 Humble、MoveIt 2、colcon 可用。
3. 确认 Windows `.venv-gemini` 可运行相机和舵机代码。
4. 确认舵机 COM 口、相机 USB 口、供电方式。
5. 建立硬件参数记录表。

输出：

- `使用手册.md`
- `机械臂开发规划.md`
- `config/joint_servo_map.yaml`
- `config/robot_dimensions.yaml`

验收：

- `ros2 run demo_nodes_cpp talker` 正常。
- `colcon build` 正常。
- `python project\src\bus_servo.py status COMx 1` 能读到状态。
- `python project\src\gemini_windows\red_block_detector.py` 能检测目标。

### 阶段 1：单关节标定和安全限位

目标：确认每个舵机对应哪个机械关节，以及舵机值和关节角度的关系。

任务：

1. 单独读取每个舵机 ID。
2. 给每个舵机发送小幅移动命令，例如当前位置附近 ±20。
3. 记录关节运动方向。
4. 记录安全位置范围。
5. 建立 `servo_position <-> joint_angle` 映射。
6. 加入软件限位，禁止越界命令下发。

输出：

- `config/joint_servo_map.yaml`
- 单关节测试记录
- 安全限位表

验收：

- 每个关节都能独立低速运动。
- 任意命令超过安全范围时会被拒绝。
- 可以一键回到安全初始姿态 `home`。

### 阶段 2：机械臂模型和正运动学

目标：建立可计算的六轴机械臂模型。

任务：

1. 测量连杆长度。
2. 确定每个关节旋转轴方向。
3. 编写 URDF/Xacro 初版。
4. 在 RViz 显示机械臂模型。
5. 编写或验证正运动学：`joint_angles -> tcp_pose`。
6. 将舵机位置映射为关节角度。

输出：

- `jetarm_description`
- `urdf/jetarm.urdf.xacro`
- `config/joint_limits.yaml`
- RViz 显示配置

验收：

- RViz 中关节方向和真机一致。
- 给定一组关节角，模型末端位置和实际机械臂大致一致。
- 关节限位在 RViz 中有效。

### 阶段 3：逆运动学和基础轨迹

目标：让机械臂能到达指定空间点。

任务：

1. 实现或接入 IK 求解。
2. 先只做固定姿态抓取，例如末端垂直向下。
3. 输入目标 `x, y, z`，输出六轴关节角。
4. 加入关节空间插补。
5. 轨迹执行前检查限位、速度和高度。

第一版可以先做简化：

```text
base yaw 由目标 x/y 决定
shoulder + elbow + wrist 解平面二连杆
wrist yaw/roll 固定或跟随夹爪方向
```

如果几何结构复杂，再接入成熟 IK 插件或 MoveIt 2。

输出：

- `jetarm_control` 初版
- `jetarm_motion` 或 `jetarm_task` 中的基础 IK/轨迹模块
- 固定点位动作脚本

验收：

- 输入 3-5 个固定空间点，机械臂能低速移动到目标上方。
- 轨迹过程中不撞桌面、不撞自身明显结构。
- 无目标时能回 `home`。

### 阶段 4：MoveIt 2 接入

目标：用 MoveIt 2 管理运动规划、碰撞检测和 RViz 调试。

任务：

1. 用 MoveIt Setup Assistant 生成 `jetarm_moveit_config`。
2. 设置 planning group，例如 `arm` 和 `gripper`。
3. 设置自碰撞矩阵。
4. 添加简化碰撞物体：桌面、底座、相机支架、放置区。
5. 测试 RViz 中拖动末端规划。
6. 将 MoveIt 输出的轨迹转换为舵机位置命令。

输出：

- `jetarm_moveit_config`
- `move_group` 可启动
- RViz MotionPlanning 面板可规划

验收：

- RViz 中能够规划到多个目标姿态。
- MoveIt 能识别明显不可达或碰撞目标。
- 轨迹可以低速下发到真机。

### 阶段 5：视觉定位和手眼标定

目标：把相机识别到的目标位置转换到机械臂坐标系。

任务：

1. 固定相机安装方式。
2. 读取/标定相机内参。
3. 识别标定板或已知坐标点。
4. 求解 `camera_frame -> base_link` 或 `camera_frame -> tool0`。
5. 将 `red_block_detector.py` 的输出改为发布目标坐标。
6. 加入桌面平面约束，减少深度噪声。

输出：

- `config/camera_intrinsics.yaml`
- `config/hand_eye.yaml`
- `jetarm_vision` 目标发布节点

验收：

- 相机坐标转换到机械臂基坐标后，误差在可接受范围内。
- 同一个静止目标重复测量，坐标波动较小。
- 机械臂移动到目标上方时，横向偏差可通过简单补偿修正。

### 阶段 6：固定流程抓取

目标：完成不依赖自然语言的规则式抓取。

任务：

1. 目标检测只选择红色方块。
2. 根据目标坐标生成抓取姿态。
3. 生成动作序列：

```text
home
-> observe
-> pre_grasp
-> descend
-> close_gripper
-> lift
-> place_pre
-> place
-> open_gripper
-> home
```

4. 每一步执行后读取舵机状态。
5. 失败时停止并回安全位。

输出：

- `jetarm_task` 固定抓取节点
- 抓取动作配置
- 成功率测试记录

验收：

- 单个红色方块抓取成功率达到可演示水平。
- 连续抓取 10 次，失败时不发生危险运动。
- 能保存视觉和执行日志用于复盘。

### 阶段 7：自然语言任务接入

目标：让自然语言只负责“选择目标和任务”，不绕过规划和安全层。

任务：

1. 定义任务 JSON schema。
2. 支持命令：
   - 抓取红色方块
   - 把红色方块放到左侧区域
   - 回到初始位置
   - 停止
3. LLM 输出只允许结构化 JSON。
4. JSON 进入任务规划器前做白名单和参数检查。
5. 危险命令必须拒绝。

任务结构示例：

```json
{
  "intent": "pick_and_place",
  "target": {
    "color": "red",
    "shape": "block"
  },
  "destination": {
    "type": "zone",
    "name": "left_area"
  },
  "constraints": {
    "speed": "slow",
    "confirm_before_execute": true
  }
}
```

输出：

- `jetarm_task` 自然语言解析接口
- 任务白名单
- 安全拒绝规则

验收：

- 正常命令能转成结构化任务。
- 未知目标、危险动作、越界区域会被拒绝。
- 舵机控制层不直接暴露给 AI。

### 阶段 8：DK2500 部署

目标：把 Windows/WSL 验证过的逻辑迁移到 DK2500。

任务：

1. 在 DK2500 上安装 Ubuntu 22.04。
2. 安装 ROS 2 Humble。
3. 部署 `~/ros2_ws`。
4. 配置相机驱动和串口权限。
5. 固定设备名，例如 `/dev/jetarm_servo`。
6. 配置一键 launch。

输出：

- DK2500 部署说明
- `jetarm_bringup`
- systemd 或手动启动脚本

验收：

- DK2500 上能启动相机、控制、规划和任务节点。
- 不依赖 Windows 电脑完成一次固定抓取。

## 8. 测试计划

### 8.1 单元测试

| 测试 | 目标 |
|---|---|
| 舵机协议测试 | 确认组帧、校验、解析不出错 |
| 关节映射测试 | 确认角度到舵机位置转换正确 |
| 限位测试 | 确认越界命令被拒绝 |
| 坐标转换测试 | 确认相机坐标到基坐标转换正确 |
| IK 测试 | 确认固定目标能得到合理关节角 |

### 8.2 硬件测试

| 测试 | 目标 |
|---|---|
| 单舵机小幅运动 | 验证 ID、方向、限位 |
| 多关节回 home | 验证初始姿态可重复 |
| 固定点移动 | 验证 IK 和轨迹执行 |
| 目标上方移动 | 验证视觉定位和坐标转换 |
| 空抓取流程 | 不放物体，验证动作序列安全 |
| 真实抓取流程 | 放置物体，验证闭环效果 |

### 8.3 验收指标

第一阶段建议指标：

| 指标 | 建议目标 |
|---|---|
| 相机目标重复定位误差 | 小于 `10 mm` 起步，后续优化 |
| 固定点到达误差 | 小于 `15 mm` 起步 |
| 单目标抓取成功率 | 先达到 `60%`，再优化到 `80%+` |
| 单次任务平均耗时 | 先不追求速度，保证安全 |
| 异常停止 | 任意异常必须能停住或回安全位 |

## 9. 安全策略

### 9.1 执行前检查

每次执行动作前必须检查：

- ROS/控制程序是否在线。
- 舵机状态是否可读。
- 电压是否正常。
- 温度是否过高。
- 目标点是否在工作空间内。
- 关节角是否在限位内。
- 轨迹是否低于速度限制。
- 夹爪和桌面是否有碰撞风险。

### 9.2 软件限位

至少三层限位：

1. 舵机位置值限位：防止发出超出 `0..1000` 或安全区间的命令。
2. 关节角限位：防止 IK 输出机械上不可达角度。
3. 工作空间限位：防止末端进入危险区域，例如桌面以下。

### 9.3 急停和恢复

必须准备：

- 软件停止命令。
- 电源急停方式。
- 失控时断电流程。
- 停止后回 home 的恢复流程。

第一版调试时建议：
