# 环境重建说明

本文用于在新机器上重新创建当前 JetArm RGB 视觉闭环开发环境。

## 1. Windows Python 环境

推荐在 `D:\jetarm` 下创建虚拟环境：

```powershell
cd D:\jetarm
py -3.12 -m venv .venv-jetarm
.\.venv-jetarm\Scripts\activate
python -m pip install --upgrade pip
pip install -r project\requirements.txt
```

验证基础依赖：

```powershell
python -c "import cv2, numpy, serial; print('ok')"
```

## 2. ROS2 Humble 环境

Ubuntu 22.04 中使用 ROS2 Humble：

```bash
source /opt/ros/humble/setup.bash
mkdir -p ~/ros2_ws/src
cp -r /mnt/d/jetarm/project/ros2_ws/src/jetarm_description ~/ros2_ws/src/
chmod +x ~/ros2_ws/src/jetarm_description/scripts/joint_state_mapper.py
cd ~/ros2_ws
colcon build --packages-select jetarm_description
source install/setup.bash
ros2 launch jetarm_description display.launch.py
```

如果缺少 GUI：

```bash
sudo apt update
sudo apt install -y ros-humble-joint-state-publisher-gui
```

## 3. 项目测试

Windows 下从 `D:\jetarm` 运行：

```powershell
python -m unittest discover -s project\tests
```

## 4. 当前环境重点

| 链路 | 要求 |
|---|---|
| RGB 图像处理 | OpenCV 可用 |
| 舵机串口 | `pyserial` 可用，能打开控制板 COM 口 |
| ROS2 显示 | `jetarm_description` 能在 RViz 中显示 |
| J6 映射 | GUI 原始范围 `0..1000` 能映射为夹爪张开/闭合显示 |

## 5. 故障检查

| 现象 | 检查 |
|---|---|
| Python 找不到依赖 | 确认虚拟环境已激活，并重新执行 `pip install -r project\requirements.txt` |
| 串口打不开 | 检查 COM 口编号、USB 转串口驱动、控制板供电 |
| ROS2 找不到包 | 确认已复制 `jetarm_description` 到 `~/ros2_ws/src` 并执行 `colcon build` |
| RViz 没有夹爪动作 | 确认 `joint_state_mapper.py` 有执行权限，并通过 `display.launch.py` 启动 |
