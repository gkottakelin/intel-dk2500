# 环境重建说明

> 本文用于替代直接上传 `.venv-gemini/`。其他电脑或 DK2500 迁移时，应按本文重新创建环境。

## 1. Windows 开发环境

当前 Windows 阶段建议使用 Python 3.12。

在 `D:\jetarm` 下创建虚拟环境：

```powershell
cd D:\jetarm
py -3.12 -m venv .venv-gemini
.\.venv-gemini\Scripts\activate
python -m pip install --upgrade pip
pip install -r project\requirements.txt
```

验证环境：

```powershell
python project\src\gemini_windows\diagnose_environment.py
python project\src\gemini_windows\windows_usb_check.py
```

## 3. 舵机开发环境

舵机控制代码需要串口库：

```powershell
pip install pyserial
```

测试舵机状态读取：

```powershell
python project\src\bus_servo.py status COM3 1
```

测试 Servo 模式：

```powershell
python project\src\bus_servo.py servo COM3 1 500 1000
```

测试 Motor 模式：

```powershell
python project\src\bus_servo.py motor COM3 1 200
```

其中：

| 参数 | 含义 |
|---|---|
| `COM3` | 按实际串口修改 |
| `1` | 舵机 ID |
| `500` | 目标位置 |
| `1000` | 运行时间，单位 ms |
| `200` | motor 模式速度 |

## 4. Gemini Pro Plus Windows 调试环境

如果只验证 Windows 上位机相机图像，可先使用 Orbbec Viewer。

如果运行项目脚本：

```powershell
python project\src\gemini_windows\windows_usb_check.py
python project\src\gemini_windows\run_openni_sample.py depth
python project\src\gemini_windows\run_openni_sample.py color-uvc
python project\src\gemini_windows\opencv_uvc_color_test.py --index 0
```

注意：

| 问题 | 处理 |
|---|---|
| Orbbec Viewer 正在运行 | 关闭 Viewer 后再运行 SDK/OpenNI 示例 |
| `list_devices.py` 显示设备数量为 0 | 说明 pyorbbecsdk2 没有枚举到设备，不代表 Windows 没识别相机 |
| `ColorReaderPoll.exe` 报错 | Gemini Pro Plus 的彩色流优先走 UVC 路线 |
| 深度图可用但彩色图不可用 | 使用 OpenNI 读深度，使用 UVC/OpenCV 读彩色 |

## 5. DK2500/ROS2 迁移环境

DK2500 阶段建议迁移到 Ubuntu + ROS2。

迁移时不复用 Windows `.venv-gemini`，而是重新安装：

| 环境 | 建议 |
|---|---|
| 操作系统 | Ubuntu |
| ROS2 | 按 DK2500 实际 Ubuntu 版本选择 Humble 或 Jazzy |
| 相机驱动 | OrbbecSDK ROS2 wrapper |
| 舵机控制 | Python 串口节点或 ROS2 控制节点 |
| AI 模块 | 可先远程调用模型，后续再考虑本地推理 |

Windows 阶段的代码作用是验证相机、舵机和算法思路；DK2500 阶段需要重新确认 USB、串口、ROS2 话题和驱动兼容性。
