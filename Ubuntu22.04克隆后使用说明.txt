JetArm 项目 Ubuntu 22.04 使用说明
================================

一、项目简介
------------

本项目包含：

1. JetArm 机械臂串口控制程序
2. Ubuntu 22.04 操作终端
3. Gemini RGB-D 相机程序
4. AI 自然语言控制终端
5. MCP 工具服务
6. ROS 2 Humble 机械臂模型与 RViz 显示
7. 单元测试和运动学验证代码

Ubuntu 22.04 推荐使用 Python 3.10。

注意：Ubuntu 下不要直接安装根目录的 requirements.txt。该文件包含
Windows/Python 3.12 相关依赖。Ubuntu 主要使用：

- requirements-ai.txt
- ubuntu22_04_operation_terminal/
- ubuntu22_04_gemini_camera/


二、克隆项目
------------

安装 Git：

    sudo apt update
    sudo apt install -y git

使用 HTTPS 克隆：

    git clone https://github.com/gkottakelin/intel-dk2500.git
    cd intel-dk2500

如果已经配置 GitHub SSH，也可以使用：

    git clone git@github.com:gkottakelin/intel-dk2500.git
    cd intel-dk2500

确认当前目录正确：

    ls

应能看到以下目录或文件：

    src
    config
    tests
    ubuntu22_04_operation_terminal
    ubuntu22_04_gemini_camera
    requirements-ai.txt
    README.md


三、安装基础依赖
----------------

    sudo apt update
    sudo apt install -y \
        python3 \
        python3-venv \
        python3-tk \
        python3-numpy \
        python3-serial \
        fonts-noto-cjk

将当前用户加入串口用户组：

    sudo usermod -aG dialout "$USER"

执行后注销并重新登录，或者临时执行：

    newgrp dialout

检查串口设备：

    ls /dev/ttyUSB* /dev/ttyACM* 2>/dev/null
    ls -l /dev/serial/by-id/ 2>/dev/null

推荐使用 /dev/serial/by-id/ 下的稳定设备路径，避免 USB 设备编号变化。


四、运行 Ubuntu 机械臂操作终端
------------------------------

4.1 创建运行环境

    cd ubuntu22_04_operation_terminal
    bash setup.sh

该脚本会创建：

    ubuntu22_04_operation_terminal/.venv/

4.2 首次运行模拟模式

首次运行建议先使用模拟模式，不连接真实机械臂：

    bash run.sh --dry-run

模拟模式可用于检查界面和控制逻辑。

4.3 查看串口

    bash run.sh --list-ports

如果没有检测到串口，运行：

    bash run.sh --diagnose-ports

4.4 连接真实机械臂

指定串口运行：

    bash run.sh --port /dev/ttyUSB0

推荐使用稳定路径：

    bash run.sh --port /dev/serial/by-id/usb-你的设备名称

不指定 --port 时，程序会弹出串口选择界面：

    bash run.sh

操作终端配置文件：

    ubuntu22_04_operation_terminal/config/terminal.json

该文件包含串口、波特率、舵机 ID、关节限位和 Home 姿态等参数。


五、配置和运行 AI 自然语言控制终端
------------------------------------

5.1 创建 AI 虚拟环境

在项目根目录执行：

    cd /path/to/intel-dk2500
    python3 -m venv .venv-ai
    source .venv-ai/bin/activate
    python -m pip install --upgrade pip
    python -m pip install -r requirements-ai.txt

以后重新打开终端时，需要重新激活环境：

    cd /path/to/intel-dk2500
    source .venv-ai/bin/activate

5.1.1 启动JetArm总控终端

总控终端不会修改现有工作流。按下功能键后，它只会打开新的独立终端，
并执行Git Pull、机械臂控制run.sh、相机显示run.sh、人工测试模块V2或Agent
的现有入口。

在项目根目录执行：

    chmod +x run_control_center.sh
    bash run_control_center.sh

配置中心可以编辑接口、抓取点和Agent接口配置。机械臂参数页面只读取并
显示ubuntu22_04_operation_terminal/config/terminal.json，不提供修改或保存
机械臂参数的功能。

5.2 配置 API Key

复制环境变量模板：

    cp .env.example .env
    nano .env

填写以下内容：

    MOONSHOT_API_KEY=你的Kimi_API_Key
    JETARM_API_BASE_URL=https://api.moonshot.cn/v1
    JETARM_API_MODEL=kimi-k2.6

设置文件权限：

    chmod 600 .env

不要将真实 API Key 提交到 Git。

5.3 运行 AI 对话终端

    python3 -m src.jetarm_agent

只发送一条消息后退出：

    python3 -m src.jetarm_agent --once "你好，请介绍一下你能做什么"

推荐先在接口与抓取点配置界面中保存抓取点像素：

    python3 -m src.jetarm_agent.device_config

抓取点会与机械臂串口和 Gemini 相机配置一起保存。需要在当前运行期间
临时重新标定时，再在交互模式输入：

    /grasp-point 320 147

然后再用自然语言要求抓取，例如“抓取红色物块”。Agent 自行判断是否为
抓取意图；进入抓取后识别目标物品，并进行3×3、四层数据分块定位。
每层只选择目标中心所在分块，分块图不绘制标签；程序把最终分块中心换算为
原图像素并强制传给控制器。控制端使用人工测试 V2
工作流（有效进展检测关闭）执行运动，每次运动结束后自动重新取图，再由
Agent 重新识别目标并重做四层分块，直到完成最终下降和夹取；程序确认 J6 位置稳定后
才返回 Home。
Home 后程序会自动返回新图；Agent 必须根据新图确认实际抓取结果，确认失败
会继续闭环，只有确认成功后才报告抓取完成。

最终target_x/target_y由四层分块边界中心产生，坐标固定使用原始RGB图像：
左上角为(0,0)、X向右、Y向下，右下角为(width-1,height-1)。禁止使用
左下角原点、Y向上、百分比、缩放后或翻转后的坐标。控制端会按最新原图
宽高检查目标点，越界时拒绝运动。

单次 `--once` 抓取可在启动参数中输入抓取点：

    python3 -m src.jetarm_agent \
        --agent-grasp-x 320 \
        --agent-grasp-y 147 \
        --once "抓取红色物块"

5.4 运行 AI 工具调用测试

该测试需要 API Key，但不会访问相机和真实机械臂：

    python3 -m src.jetarm_agent --tool-test

运行本地自动化测试：

    python3 -m unittest \
        tests.test_ai_agent \
        tests.test_ai_arm_control \
        tests.test_ai_camera \
        tests.test_ai_mcp


六、启动人工像素闭环测试模块
------------------------------

人工测试模块不是操作终端，启动入口是 AI 命令行模块的
--manual-pixel-test 参数，对应源码：

    src/jetarm_agent/cli.py

该模式具有以下特点：

1. 不调用 AI API。
2. 不连接相机。
3. 由人工输入目标点像素坐标。
4. 使用模拟图像和当前机械臂状态模拟视觉闭环。
5. 默认使用 dry-run，不会驱动真实机械臂。

抓取点需要区分为两类配置：

1. 抓取点像素坐标：人工测试中用于和目标点像素比较的参考点。
2. 机械臂物理抓取点：用于正向运动学计算的 TCP/夹爪位置。

人工测试的抓取点像素坐标通过以下参数配置：

    --manual-grasp-x 抓取点像素X
    --manual-grasp-y 抓取点像素Y

如果不指定，默认使用模拟图像中心：

    抓取点X = manual_image_width / 2
    抓取点Y = manual_image_height / 2

例如，模拟图像为 640x480，抓取点为 (320,240)：

    python3 -m src.jetarm_agent \
        --manual-pixel-test \
        --arm-mode dry-run \
        --manual-image-width 640 \
        --manual-image-height 480 \
        --manual-grasp-x 320 \
        --manual-grasp-y 240

机械臂物理抓取点配置位于：

    ubuntu22_04_operation_terminal/config/terminal.json

重点检查或修改以下字段：

    "geometry_m": {
      "base_to_joint2": 0.1,
      "joint2_to_joint3": 0.13,
      "joint3_to_joint4": 0.13,
      "joint4_to_joint5": 0.06,
      "joint5_to_tcp": 0.1
    }

其中 joint5_to_tcp 表示 J5 到物理抓取点/TCP 的距离，默认值为 0.1 米。
camera_control 中的 grasp_to_camera_* 字段用于相机与抓取点之间的相对位置，
修改前应重新测量相机安装位置。

注意：config/camera_structure.yaml 当前主要保存相机安装和标定信息，
其中相机内参、外参仍有待标定；它不是人工测试抓取点像素的配置入口。

在项目根目录执行：

    source .venv-ai/bin/activate
    python3 -m src.jetarm_agent \
        --manual-pixel-test \
        --arm-mode dry-run

启动后，程序会提示输入目标点像素，例如：

    目标点像素 x y: 450 230

或者：

    目标点像素 x y: 450,230

输入 q 退出人工测试。

默认模拟图像大小为 640x480，抓取点默认为图像中心。
也可以手动指定模拟图像和抓取点：

    python3 -m src.jetarm_agent \
        --manual-pixel-test \
        --arm-mode dry-run \
        --manual-image-width 640 \
        --manual-image-height 480 \
        --manual-grasp-x 320 \
        --manual-grasp-y 240

人工测试 V2 使用独立入口：

    python3 -m src.jetarm_agent \
        --manual-pixel-test-v2 \
        --arm-mode dry-run

V2 复用原人工测试的逐轮像素输入、像素比例、动态容差、分段下降、
最终抓取和 Home 工作流，但运动程序改用 camera_vector_terminal_v2.py
中的 CameraVectorV2Runtime。V2 默认抓取点像素为：

    (320,147)

动态容差按X、Y轴独立判断，边界高度归入容差更小的低高度档：

    Z > 15cm：       X=30px，Y=18px
    10cm < Z <= 15cm：X=25px，Y=15px
    5cm < Z <= 10cm： X=18px，Y=12px
    Z <= 5cm：        X=15px，Y=10px

仍可使用 --manual-grasp-x 和 --manual-grasp-y 显式覆盖。

有效进展检测默认开启。可使用以下参数显式打开或关闭：

    --manual-progress-check on
    --manual-progress-check off

关闭后，已经实际下发运动的进展不足或反馈未达标仍会输出，但不再作为
人工像素闭环的中止条件。规划被拒绝、未实际下发运动、串口异常以及最终
抓取高度未安全到达等保护不会被关闭。

V2 默认严格保持摄像头—抓取点连线倾角。仅在下降动作因严格姿态约束
触及关节限位、目标不可达或无法继续运动时，自动改用放宽倾角的后备
逆解；前后左右和上升不会放宽。每次发生放宽时，终端会输出
“姿态约束=已放宽”，并附带原因和放宽步数。最终高度使用 V2 的
4 mm 位置容差。

V2 的 forward/backward 以实际抓取点 XYZ 坐标为唯一基准：forward 使抓取点 Y
减小，backward 使抓取点 Y 增加；left 使 X 减小，right 使 X 增加；up 使 Z 增加，
down 使 Z 减小。六方向运动都保持摄像头—抓取点姿态。请求点因关节限位不可达时，
程序前往本次目标方向上距离请求点最近的可达位置，并在输出状态中标明。
人工测试 V2 不再自行执行机械臂
运动，而是把方向、速度和持续时间交给 run_camera_vector_v2 的共享终端
输入路径。例如左移 2.5 cm、速度 1 cm/s，会转换为：

    bash run_camera_vector_v2.sh \
        --dry-run \
        --motion-direction left \
        --motion-speed-cm-s 1 \
        --motion-duration-s 2.5

真实硬件使用同一组 motion 参数，并增加 --port 指定串口；不要添加
--dry-run。

V2 硬件模式示例：

    python3 -m src.jetarm_agent \
        --manual-pixel-test-v2 \
        --arm-mode hardware \
        --arm-port /dev/serial/by-id/usb-你的设备名称

如果要让人工测试模块驱动真实机械臂，必须明确指定 hardware，
并使用稳定的串口路径：

    python3 -m src.jetarm_agent \
        --manual-pixel-test \
        --arm-mode hardware \
        --arm-port /dev/serial/by-id/usb-你的设备名称

硬件模式启动后，程序会要求输入 RUN 才开始运行；输入其他内容会取消。
真实硬件测试前必须清空工作空间，并准备随时断电。


七、配置真实机械臂和相机
------------------------

首次使用 AI 控制机械臂时，运行设备配置程序：

    source .venv-ai/bin/activate
    python3 -m src.jetarm_agent.device_config

程序可以配置：

1. 机械臂模式
2. 机械臂串口
3. Gemini 相机序列号或 UID
4. 抓取点像素 X、Y
5. 操作终端配置文件

配置结果保存到：

    config/devices.json

该文件是本机配置文件，默认不会提交到 Git。

如果暂时不连接真实硬件，可以使用无界面模拟配置：

    python3 -m src.jetarm_agent.device_config \
        --no-gui \
        --arm-mode dry-run \
        --grasp-point-x 320 \
        --grasp-point-y 147

配置完成后启动 Agent：

    python3 -m src.jetarm_agent

常用交互命令：

    /help
    /tool-test
    /camera
    /workflow
    /arm-status
    /arm-stop
    /arm-home
    /arm-init
    /exit

`/arm-init` 会先让 J1-J5 回到 Home，再把 J6 张开到 400，并重置抓取闭环。


八、运行 Gemini 相机程序
------------------------

Gemini 相机程序要求：

1. Ubuntu 22.04
2. Intel/AMD x86_64
3. USB 3.0
4. 桌面环境或可用的图形显示
5. Orbbec Gemini 相机

安装依赖：

    cd ubuntu22_04_gemini_camera
    sudo apt update
    sudo apt install -y \
        python3 \
        python3-venv \
        python3-tk \
        python3-numpy \
        python3-opencv \
        libusb-1.0-0 \
        usbutils \
        fonts-noto-cjk

创建环境：

    bash setup.sh

安装 USB 权限规则：

    sudo bash install_udev_rules.sh

拔出并重新插入相机，然后检查设备：

    lsusb | grep -i 2bc5
    bash run.sh --diagnose

列出相机：

    bash run.sh --list-devices

使用第一个相机：

    bash run.sh --first-device

指定相机序列号：

    bash run.sh --serial 你的相机序列号

查看相机内参：

    bash run.sh \
        --serial 你的相机序列号 \
        --read-intrinsics

启动相机查看器：

    bash run.sh

查看器操作：

- 鼠标点击彩色图可以查看对应位置的深度。
- 按 s 保存图像和深度数据。
- 按 q 或 Esc 退出。


九、ROS 2 Humble 模型显示
--------------------------

如果已经安装 ROS 2 Humble，先加载环境：

    source /opt/ros/humble/setup.bash

安装显示相关组件：

    sudo apt install -y \
        ros-humble-desktop \
        ros-humble-joint-state-publisher-gui

构建 ROS 2 工作区：

    cd ros2_ws
    colcon build --packages-select jetarm_description
    source install/setup.bash

启动 RViz：

    ros2 launch jetarm_description display.launch.py

不使用关节滑块界面：

    ros2 launch jetarm_description display.launch.py use_gui:=false

十一、推荐启动顺序
----------------

第一次部署建议按以下顺序执行：

1. 安装 Ubuntu 基础依赖。
2. 克隆项目并进入项目根目录。
3. 运行机械臂操作终端的 dry-run 模式。
4. 运行完整单元测试。
5. 安装并检查串口权限。
6. 单独运行 Gemini 相机诊断程序。
7. 连接真实机械臂，读取设备状态。
8. 确认 Home 姿态、舵机 ID 和关节限位。
9. 最后再启动 AI 自然语言控制。


十二、安全注意事项
--------------------

1. 真实机械臂运行前必须清空工作空间。
2. 首次运行必须先使用 --dry-run。
3. 确认机械臂串口、舵机 ID、运动方向和限位。
4. 机械臂运动时人员不要靠近工作范围。
5. 必须准备可以立即断电的电源开关。
6. 不要使用 sudo 运行机械臂程序。
7. 修改关节限位或 Home 姿态后，必须重新运行测试。


十三、常见问题
--------------

1. 串口权限不足

错误类似：

    Permission denied: /dev/ttyUSB0

解决方法：

    sudo usermod -aG dialout "$USER"

注销并重新登录后再运行程序。

2. 找不到串口

    bash ubuntu22_04_operation_terminal/run.sh --list-ports
    bash ubuntu22_04_operation_terminal/run.sh --diagnose-ports

检查 USB 线、CH340 驱动、brltty 占用情况，以及机械臂是否已经上电。

3. 找不到 Gemini 相机

    lsusb | grep -i 2bc5
    bash ubuntu22_04_gemini_camera/run.sh --diagnose

确认已经安装 udev 规则，并关闭 Orbbec Viewer 等可能占用相机的程序。

4. AI API Key 无效

检查 .env 文件是否存在，并确认变量名称为：

    MOONSHOT_API_KEY

同时确认已经激活 .venv-ai 环境。

5. 没有图形界面

机械臂操作终端、设备配置程序和 Gemini 查看器都需要图形显示环境。
Ubuntu Server 或 SSH 环境需要配置桌面、X11 转发或其他远程显示方案。
