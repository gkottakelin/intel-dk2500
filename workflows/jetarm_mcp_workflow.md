# JetArm MCP 工作流规范

## 适用范围

用户明确要求机械臂移动、回到 Home、停止、旋转腕部或控制夹爪时使用。本工作流只配置
一路 RGB 相机，不启动或依赖深度流。

## 强制执行流程

1. 读取用户原始指令，不补充用户没有要求的动作。
2. 提取方向、总距离和可选速度。距离缺失时必须询问，禁止猜测。
3. 把方向和厘米数转换为紧凑命令：`前N`、`后N`、`左N`、`右N`、`上N`、`下N`。
4. 未指定速度时使用 `1.5 cm/s`；用户指定速度必须在 `1–5 cm/s`。
5. 调用 MCP 工具 `move_jetarm(command, speed_cm_s)`。禁止绕过 MCP 直接操作串口。
6. 控制程序把总距离拆成若干不超过 `3 cm` 的物理分段，顺序执行全部分段。
7. 工具返回后检查 `status`：
   - `status=ok`：可以报告动作完成，并说明总距离、速度和分段数。
   - `status=error`：不得声称完成；报告错误，并在存在运动风险时调用 `stop_jetarm`。
8. 最终总结必须以 MCP 的实际返回值为依据，不得虚构位置、视觉结果或完成状态。

## 示例

用户输入：`向前5厘米`

```text
自然语言：向前5厘米
解析：方向=前，总距离=5cm，速度=默认1.5cm/s
MCP调用：move_jetarm(command="前5", speed_cm_s=1.5)
控制分段：前3 → 前2
MCP返回：status=ok, segment_count=2
Agent总结：机械臂已按1.5cm/s向前移动5cm，共执行2个分段。
```

## 安全约束

- 单个物理分段不得超过 `3 cm`，总距离不得超过 MCP 服务配置的上限。
- 不允许小于 `1 cm/s` 或大于 `5 cm/s`。
- 方向、距离不明确时不执行。
- 用户要求停止时立即调用 `stop_jetarm`，不得等待当前对话规划结束。
- RGB 相机未实际取帧时，禁止声称已经看见、识别或定位物体。
