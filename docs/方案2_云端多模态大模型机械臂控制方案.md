# 方案2：云端多模态大模型机械臂控制方案

## 1. 方案目标

本方案目标是在主机端接入云端多模态大模型，让大模型根据 RGB 图像、视觉模块返回的数据和用户指定要求，生成可执行的结构化任务步骤。本地程序负责安全校验、视觉闭环、舵机控制和异常停止。

核心原则：

```text
云端大模型负责理解和决策
本地程序负责校验和执行
舵机底层命令只允许本地控制器生成
```

## 2. 已确认条件

| 项目 | 当前约定 |
|---|---|
| 视觉输入 | 只使用 RGB 彩色图 |
| 相机位置 | 安装在 J5/J6 附近，会随 J5 转动，并能看到夹爪 |
| 夹爪标记 | 橙色爪尖作为主定位特征，紫红色区域作为辅助特征 |
| J1-J5 | servo 模式 |
| J6 | motor 模式 |
| home | `J1=500, J2=550, J3=550, J4=900, J5=500, J6=360` |
| 桌面 | 桌面高度为 `z=0` |
| 任务结束 | 每次任务完成后回 home |

## 3. 总体架构

```text
用户自然语言
        ↓
本地采集 RGB 图像
        ↓
本地视觉模块提取视觉状态
        ↓
云端多模态大模型理解图像和任务
        ↓
返回结构化任务 JSON
        ↓
本地安全校验
        ↓
本地视觉闭环执行
        ↓
任务完成后回 home
```

## 4. 云端大模型输入

每次调用云端多模态大模型时，本地发送四类信息：

| 输入 | 内容 |
|---|---|
| 用户指令 | 例如“夹取红色物块放到蓝色框子里” |
| RGB 图像 | 当前相机看到的彩色图 |
| 本地视觉状态 | 目标候选、夹爪端点、夹爪中心线、图像坐标 |
| 机器人状态 | 当前 J1-J6 位置、home、夹爪状态、安全限制 |

推荐输入 JSON：

```json
{

  },
  "vision_state": {

    "gripper": {
      "left_tip_uv": [210, 380],
      "right_tip_uv": [430, 382],
      "center_uv": [320, 381],
      "confidence": 0.91
    },
    "objects": [
      {
        "id": "obj_1",
        "label": "red_block",
```

## 5. 云端大模型输出

云端大模型只能返回结构化任务，不允许返回舵机原始命令。

推荐输出 JSON：

```json
{
  "task_id": "task_001",
  "task_type": "pick_place",
  "language_summary": "把红色物块放到蓝色框子里",
  "target": {
    "object_id": "obj_1",
    "label": "red_block",
    "reason": "用户要求夹取红色物块"
  },
  "destination": {
    "object_id": "obj_2",
    "label": "blue_box",
    "reason": "用户要求放到蓝色框子里"
  },
  "steps": [
    {
      "action": "observe",
      "success_condition": "target_and_gripper_visible"
    },
    {
      "action": "align_gripper_to_target",
      "target_object_id": "obj_1",
      "success_condition": "target_on_gripper_centerline"
    },
    {
      "action": "grasp",
      "target_object_id": "obj_1",
      "success_condition": "j6_position_stable_2s"
    },
    {
      "action": "align_gripper_to_destination",
      "destination_object_id": "obj_2",
      "success_condition": "gripper_center_over_destination"
    },
    {
      "action": "release",

    "target_visible",

    "table_safe"
  ]

```

## 6. 本地动作白名单

本地只接受以下动作：

| 动作 | 本地执行方式 |
|---|---|
| `observe` | 读取 RGB 图，更新视觉状态 |
| `select_target` | 根据 `object_id` 选定目标 |
| `align_gripper_to_target` | 视觉闭环调整 J1-J5，让目标进入两爪中心线 |
| `grasp` | J6 motor 速度 `100` 夹取，2 秒位置不变后速度提高到 `300` |
| `align_gripper_to_destination` | 视觉闭环调整 J1-J5，对准放置目标 |
| `release` | J6 motor 速度 `-100` 张开 |
| `return_home` | J1-J6 回 home |
| `abort` | 停止动作，保持安全姿态 |

任何不在白名单内的动作直接拒绝执行。

## 7. 本地安全校验

执行云端返回任务前，本地必须检查：

1. JSON 能通过 schema 校验。
2. `task_type` 在允许范围内。
3. 每个 `action` 都在动作白名单内。
4. `target.object_id` 能在当前视觉状态中找到。
5. `destination.object_id` 能在当前视觉状态中找到。
6. 夹爪标记可见。
7. J1-J5 目标动作不会超过限位。
8. J6 只按 motor 模式抓取和释放。
9. 桌面高度约束有效。
10. 任意校验失败时执行 `abort`，不继续动作。

## 8. 执行流程

### 8.1 抓取并放置

```text
回 home
-> 读取 RGB 图
-> 本地视觉模块输出目标和夹爪状态
-> 云端大模型返回任务 JSON
-> 本地校验任务
-> 视觉闭环对准目标
-> J6 motor 夹取
-> 抬起
-> 视觉闭环对准放置位置
-> J6 motor 释放
-> 视觉确认
-> 回 home
```

### 8.2 叠放任务

```text
回 home
-> 识别待抓取目标和被叠放目标
-> 云端大模型确认任务顺序
-> 视觉闭环夹取待抓取目标
-> 视觉闭环对准被叠放目标中心
-> 逐步降低释放高度
-> 释放
-> 视觉确认叠放结果
-> 回 home
```

释放高度由视觉闭环逐步确认，不写死固定高度。

## 9. 云端提示词约束

云端大模型系统提示词应包含以下约束：

```text
你是机械臂任务规划器。
你只能根据用户指令、RGB 图像和本地视觉状态生成结构化 JSON。
你不能输出舵机原始位置、速度、串口命令或代码。
你只能从 allowed_actions 中选择动作。
你必须引用 vision_state 中存在的 object_id。
如果目标、放置区域或夹爪不可见，必须返回 abort。
每个任务最后必须包含 return_home。
```

## 10. 开发步骤

1. 定义云端输入 JSON 和输出 JSON schema。
2. 实现本地视觉状态生成器。
3. 实现云端调用模块。
4. 实现 JSON schema 校验。
5. 实现动作白名单执行器。
6. 接入 J1-J5 视觉闭环。
7. 接入 J6 motor 抓取闭环。
8. 增加失败恢复和回 home 流程。
9. 用简单任务测试：“夹取红色物块”。
10. 再测试组合任务：“把红色物块放到蓝色框子里”“把红色物块叠到黄色物块上”。

## 11. 与方案1的区别

| 对比项 | 方案1 | 方案2 |
|---|---|---|
| 决策核心 | 本地任务流程为主 | 云端多模态大模型为主 |
| 云端作用 | 解析任务、辅助目标选择 | 读取图像和视觉状态，生成完整任务步骤 |
| 本地作用 | 视觉闭环和执行 | schema 校验、安全检查、视觉闭环和执行 |
| 输出形式 | 本地规划步骤 | 云端结构化任务 JSON |
| 安全边界 | 本地控制 | 本地控制，不允许云端直接控制舵机 |

方案2适合更复杂的自然语言任务，但必须坚持本地安全校验和动作白名单。
