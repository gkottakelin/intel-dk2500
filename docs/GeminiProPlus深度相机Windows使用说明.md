# Gemini Pro Plus 深度相机 Windows 使用说明

> 适用阶段：当前在 Windows 上位机进行观察、采样、视觉算法验证。后续迁移到 DK2500/ROS2 时，再把这里验证过的数据处理逻辑改写为 ROS2 节点。

## 当前状态

从 Orbbec Viewer 截图可确认：

- 设备已被 Windows 上位机识别，设备栏显示 `SV1301S_U3 ... USB3.0`。
- 彩色图像、深度图像、红外图像均可显示。
- 当前可观察到的数据流：
  - 彩色：`640x480`，`MJPG`，`30fps`
  - 深度：`640x400`，`Y11`，`30fps`
  - 红外：`640x400`，`Y10`，`30fps`
- Viewer 中的点云、彩色、深度、红外视图用于观察和调试，真正用于抓取的是程序读取到的彩色帧、深度帧和相机内参。

## Windows 阶段的意义

Windows 上位机阶段不承担最终机械臂实时控制，主要用于：

1. 验证相机硬件是否正常出流。
2. 调整曝光、激光、分辨率、帧率等参数。
3. 观察深度图中哪些区域有效、哪些区域存在黑洞或噪声。
4. 采集 RGB-D 样本，用于后续物块识别、测距、标定。
5. 在 Python/OpenCV 中验证图像处理流程，再迁移到 ROS2。

这个阶段的核心目标是回答三个问题：

- 彩色图像能否稳定获取？
- 深度值能否稳定反映物体距离？
- 彩色像素点能否转换为机械臂可用的三维坐标？

## 推荐使用流程

### 1. 连接相机

1. 使用 USB3.0 线连接 Gemini Pro Plus。
2. 优先连接电脑的 USB3.0/USB3.2 口。
3. 打开 Orbbec Viewer。
4. 在左上角设备下拉框中确认设备被识别。
5. 打开彩色、深度、红外、点云视图。

如果画面卡顿或丢帧，优先检查：

- 是否使用 USB3.0 口和合格线材。
- 是否有其他程序占用相机。
- Viewer 中是否同时开启了过多高分辨率数据流。
- 机械臂供电和相机 USB 供电是否互相干扰。

### 2. Viewer 中观察内容

| 视图 | 作用 | 项目用途 |
|---|---|---|
| 彩色 | 观察真实画面 | 物块识别、颜色分割、目标检测 |
| 深度 | 每个像素的距离信息 | 测距、三维定位、抓取点高度估计 |
| 红外 | 观察结构光/红外纹理 | 排查深度失败原因 |
| 点云 | 观察三维空间形状 | 验证桌面、机械臂、物块空间关系 |

深度图中黑色区域通常表示无效深度，常见原因包括：

- 物体太近或太远。
- 表面反光、透明、过黑。
- 红外图案被遮挡。
- 视角太斜。
- 两路数据未对齐或边缘区域深度不稳定。

## 程序如何处理相机传回的图像数据

程序不要直接使用 Viewer 的彩色伪彩图，而应读取 SDK 输出的原始帧：

```text
Gemini Pro Plus
  -> 彩色帧 Color Frame
  -> 深度帧 Depth Frame
  -> 红外帧 IR Frame
  -> 相机内参 Camera Intrinsics
  -> 可选点云 Point Cloud
```

### 数据处理总流程

```text
采集 RGB-D 帧
  -> 时间同步
  -> 彩色图像解码
  -> 深度图转毫米/米
  -> 彩色与深度对齐
  -> 识别物块像素位置
  -> 读取该像素附近深度
  -> 像素坐标转相机三维坐标
  -> 相机坐标转机械臂基座坐标
  -> 输出抓取目标
```

### 1. 彩色图像处理

彩色数据用于识别物块，例如：

- 颜色阈值分割：适合红色、蓝色、绿色等实验物块。
- OpenCV 轮廓检测：计算目标框、中心点、面积。
- YOLO/深度学习检测：适合多类别或复杂场景。

基础处理步骤：

```text
Color Frame
  -> 解码为 BGR/RGB 图像
  -> 去畸变或裁剪 ROI
  -> HSV/模型检测
  -> 得到目标中心像素 (u, v)
```

### 2. 深度图像处理

深度数据用于测距。程序中通常以 `uint16` 形式读取原始深度，再乘以 SDK 给出的深度比例系数，得到毫米或米。

处理原则：

- 不直接信任单个像素深度。
- 在目标中心附近取一个小窗口，例如 `5x5` 或 `7x7`。
- 去掉 `0` 或无效值。
- 使用中位数作为目标深度。

示意流程：

```python
depth_roi = depth_mm[v-3:v+4, u-3:u+4]
valid_depth = depth_roi[depth_roi > 0]
z_mm = median(valid_depth)
```

如果目标中心没有有效深度，可以改用：

- 目标框内深度中位数。
- 目标框内最近的稳定深度。
- 目标上表面区域的深度。

### 3. 彩色与深度对齐

彩色分辨率是 `640x480`，深度分辨率是 `640x400`，二者不是天然一一对应。

因此有两种处理方式：

1. **启用 SDK 对齐功能**：让深度图对齐到彩色图，后续直接用彩色像素 `(u, v)` 查询深度。
2. **使用 SDK 坐标转换接口**：通过相机内参和外参把彩色点映射到深度坐标。

建议优先使用 SDK 的对齐结果。没有对齐时，不要直接拿彩色图像的 `(u, v)` 去查深度图同位置，否则会出现明显位置偏差。

### 4. 像素坐标转三维坐标

当已经得到目标像素 `(u, v)` 和深度 `Z` 后，用相机内参计算相机坐标系下的三维点：

```text
X = (u - cx) * Z / fx
Y = (v - cy) * Z / fy
Z = depth
```

其中：

- `fx, fy` 是焦距参数。
- `cx, cy` 是主点坐标。
- `Z` 使用米或毫米均可，但全流程单位必须统一。

相机坐标点记为：

```text
P_camera = [X, Y, Z, 1]
```

### 5. 相机坐标转机械臂坐标

机械臂不能直接使用相机坐标，需要通过手眼标定得到变换矩阵：

```text
P_base = T_base_camera * P_camera
```

如果相机固定在机械臂末端，是 eye-in-hand 标定；如果相机固定在桌面或支架上，是 eye-to-hand 标定。

本项目图片中相机安装在机械臂末端附近，后续优先按 eye-in-hand 思路处理：

```text
base_link
  -> shoulder/link...
  -> end_effector
  -> camera_link
  -> camera_optical_frame
```

### 6. 输出给机械臂的目标数据

视觉模块最终不应输出图像，而应输出结构化抓取目标：

```json
{
  "object_name": "red_block",
  "confidence": 0.92,
  "pixel": {"u": 320, "v": 210},
  "depth_mm": 485,
  "camera_point_mm": {"x": 12, "y": -35, "z": 485},
  "base_point_mm": {"x": 210, "y": 35, "z": 80},
  "grasp_hint": "top_grasp"
}
```

这个结果再交给运动规划模块生成：

```text
预抓取点 -> 下降 -> 夹取 -> 抬升 -> 放置点
```

## 建议先完成的三个实验

### 实验 1：固定点测距

目标：验证深度值可信。

1. 在桌面放一个方块。
2. Viewer 观察深度图。
3. 程序读取中心区域深度。
4. 用尺子测量实际距离。
5. 记录误差。

### 实验 2：颜色物块定位

目标：得到物块中心像素和深度。

1. 使用彩色图做 HSV 阈值分割。
2. 提取最大轮廓。
3. 计算中心点 `(u, v)`。
4. 在深度图读取该点附近中位数。
5. 输出 `u, v, z_mm`。

### 实验 3：三维坐标输出

目标：得到相机坐标系下的目标点。

1. 获取相机内参。
2. 使用 `(u, v, z)` 计算 `X, Y, Z`。
3. 移动方块到不同位置。
4. 观察输出坐标是否随实际移动方向一致。

## 和 ROS2 阶段的关系

Windows 阶段验证的是算法逻辑：

```text
图像采集 -> 物块识别 -> 深度测距 -> 三维点计算
```

ROS2 阶段只是把输入输出换成 ROS2 topic：

```text
Windows SDK 帧
  -> ROS2 /camera/color/image_raw
  -> ROS2 /camera/depth/image_raw
  -> ROS2 /camera/color/camera_info
```

因此，Windows 阶段写好的视觉逻辑要尽量拆成纯函数：

```text
输入：color_image, depth_image, camera_intrinsics
输出：object_pose
```

这样后续迁移到 DK2500/ROS2 时，只需要替换数据来源，不需要重写算法。

## 注意事项

- Viewer 里的彩色深度图只是可视化效果，程序应读取原始深度值。
- 深度无效值通常为 `0`，必须过滤。
- 识别目标时不要只取单点深度，建议取邻域中位数。
- 彩色图和深度图分辨率不一致时，必须对齐或做坐标映射。
- 机械臂抓取前必须完成相机到机械臂基座的标定。
- 前期测试应让机械臂低速运行，并设置软件限位和急停。
