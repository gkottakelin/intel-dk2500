# 方案3：固定物块尺寸单目测距机械臂规划方案

## 1. 方案目标

本方案用于固定尺寸物块的抓取任务。系统只读取 RGB 彩色图，通过物块在图像中的像素大小估计物块到摄像头的距离，再结合摄像头相对 J5/TCP 的安装几何关系和舵机实际位置，计算抓取点并规划机械臂动作。

核心原则：

```text
物块实际尺寸固定
-> 图像像素尺寸估计距离
-> 读取 J1-J5 舵机实际位置
-> 用实际关节位置做 FK 解算摄像头/TCP 位姿
-> 计算抓取点
-> 本地安全校验和视觉闭环执行
```

由于舵机存在误差，禁止只用“命令目标位置”计算抓取点。每次规划前必须读取舵机实际位置，并用实际位置重新计算当前机械臂状态。

## 2. 当前已知几何条件

根据当前模型和示意图：

| 参数 | 数值 | 说明 |
|---|---:|---|
| J5 到 TCP | `10.2 cm` | 当前 URDF 中 `link_5_tcp = 0.102 m` |
| 摄像头高度 | `5.5 cm` | 摄像头光心高出 J5-TCP 轴线的距离 |
| J5 到摄像头投影点 | `4.0 cm` | 图中 J5 侧到摄像头垂直投影点的距离 |
| 摄像头投影点到水平中线指向点 | `4.0 cm` | 图中虚线方向落到 J5-TCP 轴线附近的参考距离 |
| TCP 相对 J5 | `10.2 cm` | TCP 在 J5 到抓取方向的末端 |

建议定义 J5 局部坐标系：

```text
J5 原点: J5 关节中心
+X: J5 指向 TCP
+Z: 摄像头高出 J5-TCP 轴线的方向
+Y: 与图示平面垂直，按右手系定义
```

按示意图暂记：

```text
P_tcp_j5 = [0.102, 0.000, 0.000] m
C_cam_j5 = [0.040, 0.000, 0.055] m
P_centerline_j5 = [0.080, 0.000, 0.000] m
```

其中：

- `C_cam_j5` 是摄像头光心在 J5 坐标系下的位置。
- `P_centerline_j5` 是彩色图像水平中线方向在 J5-TCP 轴线附近指向的参考点。
- 虚线方向可写为：

```text
d_centerline_j5 = normalize(P_centerline_j5 - C_cam_j5)
```

这三个量必须在真机上重新测量后写入配置文件，不能只靠手绘图长期使用。

## 3. 单目测距原理

固定物块尺寸已知时，可以通过针孔相机模型估计距离。

已知：

```text
物块实际宽度: W_real
物块实际高度: H_real
图像中物块宽度: w_px
图像中物块高度: h_px
相机内参: fx, fy, cx, cy
```

距离估计：

```text
Z_from_width  = fx * W_real / w_px
Z_from_height = fy * H_real / h_px
Z_object      = robust_average(Z_from_width, Z_from_height)
```

如果物块是正方体，且可见面接近正对摄像头，可以优先使用更稳定的一条边；如果物块发生明显旋转，需要使用轮廓最小外接矩形、可见面估计或实验标定表修正。

目标中心像素 `(u, v)` 转换为摄像头坐标：

```text
X_cam = (u - cx) * Z_object / fx
Y_cam = (v - cy) * Z_object / fy
Z_cam = Z_object
P_object_cam = [X_cam, Y_cam, Z_cam]
```

注意：这里的 `Z_object` 是沿摄像头光轴方向的距离，不是桌面高度。


其中 `null` 表示必须实测，不允许猜。

## 5. 为什么必须读取舵机实际位置

实际舵机存在误差：

- 命令位置不等于实际位置。
- 多个关节误差叠加后，TCP 和摄像头位姿会明显偏移。
- 相机安装在 J5/J6 附近，会随 J5 和前面关节一起运动。

因此抓取点计算必须使用：

```text
读取 J1-J5 实际舵机位置
-> raw servo position 转 joint angle
-> 正运动学 FK
-> 得到 base_T_J5_actual
-> 得到 base_T_camera_actual
-> 得到 base_T_tcp_actual
```

禁止使用：

```text
上一次发送的目标舵机位置
-> 直接当作真实机械臂位置
```

## 6. 坐标转换流程

### 6.1 读取当前机械臂状态

```text
read_servo_status(J1)
read_servo_status(J2)
read_servo_status(J3)
read_servo_status(J4)
read_servo_status(J5)
```

得到实际舵机位置：

```text
q_raw_actual = [J1_raw, J2_raw, J3_raw, J4_raw, J5_raw]
```

再通过标定表转换为关节角：

```text
q_actual = raw_to_joint_angle(q_raw_actual)
```

### 6.2 计算实际摄像头位姿

```text
base_T_J5_actual = FK(q_actual)
J5_T_camera = pose_from_mount(C_cam_j5, d_centerline_j5)
base_T_camera_actual = base_T_J5_actual * J5_T_camera
```

### 6.3 单目估计物块坐标

```text
RGB 图像检测物块
-> 得到 bbox / contour
-> 得到 w_px, h_px, center_uv
-> 用固定物块尺寸计算 Z_object
-> 反投影得到 P_object_cam
```

### 6.4 转换到机械臂坐标

```text
P_object_base = base_T_camera_actual * P_object_cam
```

### 6.5 计算抓取点

抓取点可以先设为物块中心上方的安全点，再逐步闭环下降：

```text
P_pre_grasp_base = P_object_base + approach_offset
P_grasp_base = P_object_base + grasp_offset
```

由于当前只使用 RGB 图，最终接近时仍建议使用图像闭环确认目标位于两爪中心线。

## 7. 执行流程

### 7.1 观测

```text
回 home
-> J6 张开到 360
-> 读取 J1-J5 实际舵机位置
-> 读取 RGB 图
-> 检测物块轮廓和像素尺寸
-> 单目估计物块距离
```

### 7.2 规划

```text
用实际舵机位置做 FK
-> 计算摄像头实际位姿
-> 计算物块在 base 坐标系的位置
-> 生成预抓取点
-> 检查关节限位和桌面高度
```

### 7.3 运动

```text
J1-J5 servo 小步移动到预抓取区域
-> 每步后读取实际舵机位置
-> 重新拍图
-> 重新估计物块距离和位置
-> 修正下一步动作
```

### 7.4 抓取

```text
目标进入两爪中心线
-> J6 motor speed = 100 夹取
-> 持续读取 J6 位置
-> 连续 2s 位置不变
-> J6 motor speed = 300 短暂夹紧
-> 抬起
```

### 7.5 放置

```text
识别放置目标或区域
-> 单目估计放置目标距离
-> J1-J5 servo 小步移动
-> 视觉闭环确认位置
-> J6 motor speed = -100 张开释放
-> 回 home
```

## 8. 误差来源和修正

| 误差来源 | 影响 | 修正方法 |
|---|---|---|
| 物块实际尺寸不准 | 距离估计偏差 | 用卡尺测量物块尺寸并写入配置 |
| 相机内参不准 | 反投影坐标偏差 | 用棋盘格或标定板标定 `fx/fy/cx/cy` |
| 物块旋转 | 图像宽高不等于真实正投影 | 使用最小外接矩形、可见面判断、经验修正表 |
| 摄像头安装误差 | base 坐标整体偏移 | 实测 `C_cam_j5` 和 `P_centerline_j5` |
| 舵机实际位置误差 | TCP/相机位姿偏移 | 每步读取实际舵机位置并 FK 解算 |
| 图像检测抖动 | 距离跳变 | 多帧滤波，中位数或卡尔曼滤波 |

## 9. 安全策略

- J1-J5 每次只允许小步运动。
- 每步运动后必须重新读取实际舵机位置。
- 每步运动后必须重新拍图计算目标。
- 目标丢失时停止。
- 物块像素尺寸异常时停止。
- 计算出的物块距离超出工作范围时停止。
- 抓取点低于桌面高度 `z=0` 时停止。
- 每次任务完成后回 home。

## 10. 开发步骤

1. 测量固定物块真实尺寸。
2. 标定相机内参 `fx/fy/cx/cy`。
3. 实测摄像头在 J5 坐标系下的位置。
4. 实测彩色图像水平中线方向。
5. 建立舵机原始位置到关节角的转换表。
6. 实现物块像素尺寸检测。
7. 实现单目距离估计。
8. 实现实际舵机位置 FK 解算。
9. 实现抓取点计算。
10. 用小步 servo + 重新观测闭环验证。

## 11. 与方案1、方案2的区别

| 对比项 | 方案1 | 方案2 | 方案3 |
|---|---|---|---|
| 核心依据 | 图像误差闭环 | 云端多模态大模型任务决策 | 固定物块尺寸单目测距 |
| 距离来源 | 主要靠视觉闭环逼近 | 由本地视觉状态提供 | 由物块图像大小估计 |
| 规划基础 | 夹爪中心线和目标像素误差 | 云端结构化任务 JSON | 单目反投影 + 实际舵机 FK |
| 舵机误差处理 | 闭环修正 | 本地校验和闭环执行 | 每步读取实际位置重新解算 |
| 适合任务 | 近距离稳态抓取 | 复杂自然语言任务 | 固定尺寸物块的距离估计和抓取 |

方案3可以作为方案1的距离估计增强，也可以作为方案2的本地视觉状态输入。
