# 已有代码

### 7.1 `src/` 机械臂核心代码

| 文件                          | 功能                                                                                              | 主要入口/类                                                                         |
| --------------------------- | ----------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| `src/arm_model.py`          | 读取 `joint_servo_map.yaml`，完成原始舵机位置、舵机角、模型角之间的转换；计算 J1～J5 正运动学、TCP 位姿和数值平移雅可比；支持 JSON 命令行输出      | `JetArmKinematicModel`、`forward_kinematics_*()`、`tcp_jacobian_*()`、`main()`    |
| `src/bus_servo.py`          | 实现 `55 55 ID Length Cmd Params Checksum` 半双工串口协议；负责组帧、校验、响应解析、温度/电压/位置读取、Servo 位置模式和 Motor 速度模式 | `BusServoController`、`ServoStatus`、`build_packet()`、`parse_packet()`           |
| `src/joint_controller.py`   | 读取关节配置并把目标位置/角度转换为安全命令；按关节运动档位计算运行时间；J5/J6 可按 motor 模式运行到目标；只有 `--confirm` 才真正连接串口              | `JointCommandPlanner`、`JointCommandExecutor`、`ServoMoveCommand`、`MotorCommand` |
| `src/operation_terminal.py` | Windows Tkinter 图形界面；提供上下、水平摇杆、J5 旋转、J6 开合/抓紧、Home 和全部停止；用雅可比阻尼求解和局部搜索把 TCP 速度转换成 J1～J4 小步位置命令  | `ManualServoRuntime`、`OperationTerminalApp`、`DryRunServoController`            |
| `src/servo_limit_tester.py` | 按指定方向和速度驱动单个舵机，持续读取位置；当位置在容差内稳定指定时间时判断到达限位/接触；超时或异常时停止，并可恢复 Servo 模式                            | `ServoLimitTester`、`PositionStabilityTracker`、`LimitTestResult`                |

核心调用关系：

```text
joint_servo_map.yaml
    ├─> arm_model.py ───────────────> TCP 位姿/雅可比
    ├─> joint_controller.py ────────> 安全关节命令
    └─> operation_terminal.py ──────> 人机操作与笛卡尔小步控制

joint_controller.py / operation_terminal.py / servo_limit_tester.py
    └─> bus_servo.py ───────────────> COM 口 ─> 总线舵机
```

### 7.2 `src/gemini_windows/` 相机代码

| 文件                         | 功能                                                                           |
| -------------------------- | ---------------------------------------------------------------------------- |
| `__init__.py`              | 将目录标记为 Python 包；本身不包含业务逻辑                                                    |
| `gemini_common.py`         | pyorbbecsdk 通用工具：安全导入 SDK、选择 RGB-D 流、把彩色帧转 BGR、把深度转毫米、深度着色、局部中值深度、像素反投影和读取内参 |
| `diagnose_environment.py`  | 输出 Python、平台和工作目录；检查 NumPy、OpenCV、pyorbbecsdk，并在子进程中验证原生 SDK 是否会导入失败         |
| `list_devices.py`          | 通过 pyorbbecsdk 枚举已连接的 Orbbec/Gemini 设备                                       |
| `camera_stream_test.py`    | pyorbbecsdk RGB-D 实时测试；点击像素显示深度/三维坐标，按 `s` 保存样本                              |
| `read_intrinsics.py`       | 读取彩色和深度相机内参并以 JSON 输出                                                        |
| `opencv_uvc_color_test.py` | 用 OpenCV 探测 Windows UVC 摄像头索引，显示 Gemini 彩色画面并保存图片                            |
| `openni2_ctypes.py`        | 使用 `ctypes` 直接包装捆绑的 OpenNI2 C API；查找 DLL、枚举模式、配置激光/LDP、创建数据流和读取帧             |
| `openni_rgbd_viewer.py`    | 组合 OpenNI 深度/红外与 OpenCV UVC 彩色流；支持独立/合并窗口、镜像、曝光、激光控制、模式列表和快照保存               |
| `pointcloud_viewer.py`     | 读取 ASCII PLY 点云；降采样、深度着色、投影渲染；可监视最新 PLY，或反复启动官方点云生成器形成近实时显示                  |
| `red_block_detector.py`    | 用 HSV/轮廓检测红色物块，将彩色掩膜缩放到深度图，取有效深度中值，计算相机坐标并输出/绘制检测结果                          |
| `run_openni_sample.py`     | 统一启动捆绑的 OpenNI `.exe` 示例或本项目 Python 查看器；自动设置 DLL 搜索路径和默认参数                   |
| `windows_usb_check.py`     | 调用 PowerShell 查询 PnP/USB 设备并筛选 Orbbec、Gemini、深度相机等关键词，用于驱动和枚举排查              |
| `requirements.txt`         | 相机子目录的简化依赖说明；主环境应优先使用项目根 `requirements.txt`                                  |

### 7.3 `ubuntu22_04_gemini_camera/` Ubuntu 相机代码

| 文件 | 功能 |
|---|---|
| `gemini_camera.py` | 可视化选择 USB 相机；显示彩色/深度流；点击测距；保存彩色、深度预览和毫米深度数组；提供枚举、内参和诊断命令 |
| `orbbec_native.py` | 使用 `ctypes` 封装 Linux OrbbecSDK 1.5.7 的设备枚举、按序列号打开、RGB-D Pipeline、帧读取和内参接口 |
| `config/camera.json` | 使用与 Ubuntu 机械臂终端一致的 JSON 配置模板，保存设备序列号、超时、深度范围、镜像和输出目录 |
| `setup.sh` / `run.sh` | 检查 Ubuntu 22.04 依赖、建立虚拟环境并设置原生 SDK 动态库路径 |
| `install_udev_rules.sh` | 安装 Orbbec USB 访问规则；相机是 USB 设备，不使用机械臂的 `dialout` 串口权限 |
| `sdk/` | 从项目随附 Linux 资料复用的 Gemini 配置、udev 规则和 x86_64 OrbbecSDK 1.5.7 |
| `tests/test_gemini_camera.py` | 配置、设备选择、像素映射、深度缩放和坐标换算的无硬件测试 |
